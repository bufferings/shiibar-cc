## タスク

### T1: 右ペインの描画の文法(§4.6「描画の文法」/ conversations-design.html)

- **自分の発言 = 全幅の帯**: 右ペインの左端から右端までマージンなしで背景の明度だけを変え、
  先頭にモノスペースの `❯`(本文と同じ濃さ・セミボールドはグリフのみ)、本文と同じウェイト・
  本文 +0.5pt で発言をそのまま載せる(Markdown 描画はしない)。帯の直前は広めの余白
- **Claude の発言 = Markdown を描画**。描画範囲(§4.6): フェンスコードブロック(モノスペース +
  淡い背景 + 横スクロール・本文 −1.5pt)/ 見出し / 箇条書き・番号付きリスト / 段落 /
  インラインの code・強調・打ち消し・リンク(スタイルのみでよい)。表・引用など範囲外の構文は
  素のテキストのまま。**パースに失敗したメッセージは素のテキストにフォールバックして落ちない**
- **役割ラベルの行(You / Claude)は廃止**(帯 = 自分、地の文 = Claude)
- 本文の行長は約 60em で頭打ち(ウィンドウを広げても行が間延びしない)。行間・余白の感覚は
  conversations-design.html の CSS を目安に、システムフォントで自然に見えるよう調整してよい
- Markdown のブロック分解(フェンス分割・ブロック種別判定)は `ShiibarCcCore` に置いてテストする。
  インラインのスタイルは `AttributedString(markdown:)` 系の標準機構を使ってよい

### T2: ヒット計算を描画後テキストへ移す(§4.6)

- 語の出現計算・カウンタ・ハイライトの対象を、生テキストから**描画後のテキスト**(Markdown 記号を
  消費した後の、画面に見える文字列)に変更する。会話の絞り込みは従来どおり CLI(生テキスト)
- 畳み境界(500 文字 — §9)も**描画後テキストの文字数**で数える。切る位置はブロック構造を壊さない
  (描画済みブロック列の先頭から 500 文字分を見せる)
- ハイライトは背景色属性のみ・太字化しない(M35 と同じ理由 — 折り返しが動くとスクロール不動が壊れる)

### T3: 畳みの件数バッジと Show less(§4.6)

- 畳みの見えない部分にヒットがあるときだけ、`Show full message` の隣にハイライト色の
  `N matches` バッジ(1 件なら `1 match`)
- 展開したメッセージ(畳み閾値超のもの)の末尾に `Show less`。畳み直したら視点がそのメッセージから
  飛ばないようスクロールを寄せる。展開状態は選択の切り替えで破棄(M35 と同じ)
- ▲▼ ジャンプでの自動展開は従来どおり

### T4: ヒット目盛り(§4.6)

- 検索バー行が出ている間、右ペインの右端に会話全体のヒット分布の目盛りを重ねる。
  現在位置の目盛りだけ濃い色(ハイライトの現在位置色と同系)
- 位置は**ヒットを含むメッセージ単位の近似でよい**(メッセージの縦位置 ÷ コンテンツ全高。
  1 メッセージ内の複数ヒットの目盛りは重なってよい)。スクロールバーそのものへの描画は求めない
  (オーバーレイでよい)

### T5: 一覧の行とクローム(§4.6 / conversations-design.html)

- 行を「1 行目 = タイトル + **経過時間(右端・淡色)**/ 2 行目 = フォルダラベル(+ `· running`)」に
  変更。罫線・グループ見出しは置かない
- 行に**ホバーのハイライト**を追加(ドロップダウンと同じ作法)
- クロームを**フルハイトサイドバー**に: 信号機はサイドバーの上、サイドバー素材(沈んだ色)が
  ウィンドウ上端まで届き、右ペインは通常のウィンドウ背景。サイドバー幅の既定は 250pt(§9)
- ライト・ダーク両対応(システムの色・素材に追従 — CSS の色値をハードコードしない)

### T6: 文字サイズ(§4.6 / §4.5)

- ⌘+ / ⌘− / ⌘0 で右ペインの本文サイズを 11〜18pt(既定 13pt — §9)。コードブロックは −1.5pt 連動。
  一覧・ステータス行は固定。値は UserDefaults に永続
- Settings ウィンドウに **Conversations 節**(General / Sounds の下)を追加し、Text size の
  ステッパーを置く。ショートカットと同じ値を読み書きする(§4.5)
- ⌘ ショートカットは Conversations ウィンドウがキーのときだけ効かせる(他の窓の挙動を変えない)

### T7: 検証

- `(cd app && swift build && swift test)` 全緑。新ロジック(Markdown ブロック分解・描画後テキストの
  ヒット計算・畳み境界・隠れヒット件数・目盛り位置・サイズのクランプ)は `ShiibarCcCore` に置いて
  テストを足す
- `cargo test --workspace --locked` / `cargo clippy --all-targets --locked` 全緑(Rust 無変更の確認)
- 実機確認は所有者(スモーク項目: 帯と Markdown の読み味 / コードブロックの横スクロール /
  検索 → バッジ → 展開 → Show less / 目盛りと ▲▼ の一致 / ⌘+ / ⌘− / ⌘0 と Settings の連動 /
  一覧のホバー・時刻右端表示 / フルハイトサイドバーのライト・ダーク)
