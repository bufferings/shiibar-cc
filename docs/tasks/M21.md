## 背景とゴール

配布準備(M17〜M20)と並行して、GitHub リポジトリ側に「clone しても見えない」設定を適用した:
main / タグの保護 ruleset・Dependabot・secret scanning、および Actions 用 Secrets の登録。
これらはリポジトリの外にあるため、ドキュメント化しない限り将来の読者(あるいはリポジトリ移行)が
再現・運用できない。本 M はその記録を docs に置く。

## 内容

1. **DESIGN.md §10**: リポジトリ側の保護(main の force-push・削除禁止 / `v*` タグの削除・
   更新禁止 = §8.28 のタグ再利用禁止の強制 / Dependabot / secret scanning)を供給網対策の
   方針として 1 項目追記。詳細は DEVELOPMENT.md へ委譲
2. **DEVELOPMENT.md「リポジトリ設定と Secrets の運用」節を新設**: 各 ruleset の内容と
   一時無効化の場所 / Dependabot の挙動と PR の扱い(cargo-deny との補完関係)/
   secret scanning の限界(バイナリ・任意パスワードは対象外)/ Secrets の原本は 1Password・
   `TAP_PUSH_TOKEN` の期限(1 年)と切れたときの症状(bump-cask の認証エラー)・更新手順
