## 背景

focus の AppleScript は「select → 最後に `activate`」の順だったため、focus 元のターミナル
以外のアプリがアクティブな状態(Conversations の Jump がこの条件を日常動線にした)では、
activate のアプリ切り替えが「今の Space にあるウィンドウ」を前に出し、select 済みの
別 Space のウィンドウに遷移しない(§7-1、2026-07-17 実測)。一致を見つけたら
**activate → select の順**にすると正しく遷移する(iTerm2 で同日実証済み)。

## タスク

### T1: iterm2.rs の focus スクリプト順序

- `build_focus_script` の一致分岐を「`activate` → `tell w to select` → `tell t to select` →
  session の select → `return "FOUND"`」の順にする(§4.3)。
  **activate は一致分岐の中でのみ実行**する — 走査中・該当なし・未起動(is-running ガード)の
  経路では何も動かさない、という既存規則を変えない
- 順序を検証している構造テストを新順序(activate < select 群 < FOUND)に更新する。
  「該当なし経路に activate が無い」ことのテストが無ければ足す

### T2: apple_terminal.rs の focus スクリプト順序

- 同じ変更: 一致分岐を「`activate` → `set selected of t to true` →
  `set frontmost of w to true` → `return "FOUND"`」の順にする(selected / frontmost の
  相対順は現行のまま、activate だけを先頭へ)
- 構造テストの順序アサーションを同様に更新する
- Terminal.app での Space またぎは**未実測**(§7-7)。実装はこの順で揃え、検証は所有者スモークで
  行う — スモークで遷移しなかった場合も実装を戻さず報告する(順序を揃えること自体は
  iTerm2 の実測根拠で正当)

## テスト要求

  is-running ガードが先頭のままであること
