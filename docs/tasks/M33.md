## 背景

Conversations ウィンドウ(§4.6。索引・検索の CLI は M34、UI は M35)の Resume ボタンが呼ぶ CLI。
過去の会話を、新しい iTerm2 ウィンドウでその cwd から `claude --resume <session-id>` して再開する。
この指示書は Rust 側のみ — daemon・プロトコル・hooks には一切触れない(§8.34)。

## タスク

### T1: iterm モジュールに `open_resume_window(cwd, session_id)` を追加

- 新しい iTerm2 ウィンドウを開き、`cwd` で `claude --resume <session_id>` を実行する(§4.3)
- **focus と違い、iTerm2 が起動していなければ起動する**(新しいウィンドウを開く動詞のため)
- 起動方式(`create window with default profile command "…"` で直接実行するか、
  ウィンドウを作ってから `write text` でシェルに流すか)は**実機で確定**し、選んだ方式と理由を
  完了報告に書く。判断材料: ユーザーのシェル環境(PATH に `claude` があること)が効くか /
  ウィンドウが即閉じないか / プロファイル設定との干渉。
  **実機確認ができない環境(TCC 未許可・iTerm2 を起動できない等)の場合は `write text` 方式を
  暫定実装とし、方式の確定を所有者スモークに委ねる旨を完了報告に書く**
- **エスケープを厳密に**: cwd は空白・引用符・`$` を含み得る。AppleScript 文字列と(write text 方式なら)
  シェル引用の二重エスケープになる点に注意。session_id は UUID 想定だが検証せずエスケープで守る
- AppleScript ソースの生成は純関数として切り出し、osascript 実行部は focus と同じ差し替え機構に乗せる
  (§4.3 のテスト分離)
- osascript の TCC 権限エラーは focus と同じく他の失敗と区別して返す

### T2: `shiibar-cc resume` サブコマンド

- 形: `shiibar-cc resume --cwd <dir> <session-id>`(§4.4)。`--cwd` は必須
- **`--cwd` は絶対パスのみ受け付け、実行前にディレクトリの存在を確認する**(相対パス・不存在は
  AppleScript に触る前に exit 1、理由を stderr へ。§4.4 — transcript はフォルダより長生きするため)
- **daemon に接続しない**(iterm モジュール直呼び)。selector 解決も無い — 引数の明示指定のみ
- exit code: 0 成功 / 1 内部エラー(引数不正含む。理由は stderr)/ 3 TCC 権限エラー。
  2(該当なし)と 124 は使わない(§4.4)
- `--help` 等の体裁は既存サブコマンドに合わせる

### T3: 検証

- テスト: AppleScript ソース生成の純関数(通常 / 空白入り cwd / 引用符入り cwd / エスケープが要る
  session_id)・引数パース・exit code 系統。既存のテスト配置の流儀に従う
- `cargo test --workspace --locked` / `cargo clippy --all-targets --locked` 全緑。
  `(cd app && swift build)` が壊れていないこと(Swift 無変更のはず)
- 実機確認は所有者(スモーク項目: 実在する過去の会話の session id で新ウィンドウが開き会話が再開する /
  空白を含む cwd で動く / iTerm2 を終了した状態から動く / TCC 拒否環境で exit 3)
