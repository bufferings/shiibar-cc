## 症状と原因(調査済み)

**症状**: エージェントが 0 のときドロップダウンを開くと、パネルが「最後にエージェントが
いたときの高さ」のまま開き、小さな中身(⌄ バー + "No agents")が縦中央に浮いて、
上下にパネル背景の明るい帯が露出する。エージェントが 1 行でもいれば正常。

**原因**: M29 の panel-height 機構は、リストを `ScrollView` 内の `listHeightReader` で
実測し(`AgentListView.swift` の `naturalListHeight`)、`desiredPanelHeight` を
`AppState.setDropdownDesiredPanelHeight` に報告 → `DropdownPanelSizer` がパネル窓に
強制する、という流れ。ところが空状態の分岐は `NoAgentsRow()` を直接描くだけで
`ScrollView`(= 計測)を通らない。`desiredPanelHeight` は
`naturalListHeight > 0` ガードで nil になり、`setDropdownDesiredPanelHeight` は
nil を無視する(`AppState.swift:285` の guard)。結果、`DropdownPanelSizer.desiredHeight`
には**最後にエージェントがいたときの値が残り**、パネルが動くたびその古い高さに再強制される。

## タスク

### T1: 空状態でもパネル高さを報告する

- 空状態(grouped / flat 両分岐の `NoAgentsRow()` — 2 か所ある)でもコンテンツの
  自然高さが計測され、`desiredPanelHeight` が非 nil で報告されるようにする。
  素直な形は空状態コンテンツにも既存の `listHeightReader`(と同等の計測)を付けること —
  **新しい高さ推定定数のハードコードで代用しない**(実測が M29 bugfix の骨子)
- grouped / flat の空分岐は描くものが同一なので、変更が小さくなるなら分岐の外への
  統合をしても良い(大きくなるなら現状の 2 か所それぞれに手を入れる)
- 期待挙動:
  - エージェント 0 でドロップダウンを開く → パネル高さ = クローム + "No agents" 行の
    自然高さ。背景の帯は出ない
  - ドロップダウンを開いている間に最後のエージェントが消える → 既存の
    `onChange(of: desiredPanelHeight)` 経由でパネルが縮む
  - エージェントがいる場合の挙動(M29 の内容連動・画面キャップ)は一切変えない
- 一覧ウィンドウ側(`container.kind == .window`)は `desiredPanelHeight` の対象外
  (nil のまま)— 触らない
