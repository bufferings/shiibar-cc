## 背景

右ペインのメッセージ面を SwiftUI の Text 群から WKWebView(アプリ同梱ページ)に載せ替える(§8.38)。
検証済みのスパイク実装が土台として存在する(所有者から場所の指示を受けること。
入手できない場合は §4.6 / §8.38 を仕様として新規実装してよい)。
ヘッダ・検索バー行・Resume・一覧・Settings・⌘ズームの値の持ち方はネイティブのまま。

## タスク

### T1: ペインの本体(スパイクの productionize)

- スパイクの構成(ページテンプレート + ペイロード注入 + ブリッジ)を製品品質に引き上げる。
  規律は §4.6「描画エンジン」の段のとおり: 本文は textContent 経由のみ / nonce 付き CSP /
  ナビゲーション全キャンセル + リンクは既定ブラウザ / 非永続データストア
- **意味論は Core が正**: ヒット・畳み境界・バッジ件数・展開要否は Swift 側の既存計算を渡す。
  JS では再計算しない。オフセットは書記素 → UTF-16 変換をペイロード境界で行う(絵文字ずれ対策 —
  §8.38)
- 旧経路の削除: SwiftUI 側の MessageView・表グリッド・目盛りオーバーレイなど、ページに移った
  描画コードは消す(Core は残す)。`ConversationsTextSize` の値はそのまま CSS 変数に流す

### T2: スパイクが積み残した品質課題(§8.38 のスパイク報告より)

- **ハイライト再描画で選択が消えない**こと: クエリ打鍵ごとの全 DOM 再構築をやめ、
  ハイライトの付け替えを差分適用にする
- ヒット境界がスタイル付き run(インライン code 等)を分断したときに、padding 付きの器が
  2 つに割れて見える問題の修正(run の器の中にハイライトを入れ子にする)
- JS 例外・`evaluateJavaScript` エラーをログに出す配管(黙って壊れない)
- スクロール記憶(§4.6: 会話ごと・メッセージ単位)のページ側での実装と、下端 = 最新の既定
- ヒット目盛りは検索バー行が出ている間だけ → T6 でミニマップに置き換え

### T3: 選択とコピー(§4.6「選択とコピー」)

- ⏺ マーカー・展開/畳みボタン・その他の操作部品を選択・コピーから除外(`user-select: none` 相当)
- 右クリックメニュー: **Copy / Copy as Markdown の 2 項目を常に出す**(ショートカット併記)。
  選択が無い間は**両方 disabled**。Look Up 等それ以外の標準項目は取り除く
- **⌘C / ⇧⌘C はペイン自身の performKeyEquivalent で処理する**: このアプリは標準メニューを
  空にしている(§4.5)ため、メニュー経由のキー配達が無い
- **Copy = WebKit 標準の `copy:` を呼ぶだけ**(§4.6/§8.38(6))。書式込みのコピーは WebKit に
  任せ、自前の書き出し経路を持たない。テストは**キーが `copy:` の呼び出しに届くこと**を
  検証する(呼び出し口にスパイ。WebKit 自体を再テストしない)
- **Copy as Markdown = 選択範囲の Markdown 直列化**: 描画に使ったブロック構造とインライン
  スタイルから、選択範囲に対応する Markdown を書き戻す(見出し・リスト・コードブロック・表と、
  インラインの code・強調・打ち消し・リンクを復元。ブロックの途中で切れた選択はその部分だけ)。
  直列化は Core に置いてテストし、⇧⌘C は**合成キーイベント → 注入ペーストボードの中身**まで
  端到端で検証する
- **選択なしのコピーは全廃**: ⌘C / ⇧⌘C とも選択が無ければ何もしない(素通り)。
  カーソル下メッセージの原文コピー経路は削除する

### T4: 表示の仕上げ(§4.6 / conversations-design.html)

- 行間 1.7 / インライン code の padding(2px 5px 相当)/ 表セルは約 28em で折り返し・
  溢れる表だけ横スクロール
- ライト・ダーク: アプリの外観(Settings の Appearance 上書き込み)に追従。
  色はシステム由来(ページ側は CSS 変数で受ける)
- **強調の補完**(§4.6): 標準パーサが素のまま残した強調記号のうち、**対応の取れた組だけ**を
  強調として描画する。入れ子・対応の曖昧なケースは補完しない(誤爆より取りこぼし)。
  補完ロジックは Core に置き、日本語の句読点・括弧のケースを含めてテストする

### T5: 検索バー行・サイドバー幅・表示の不具合(§4.6 / §8.38(7)(8))

- **検索バー行を macOS 標準の文書内検索の文法に**: カウンタ `N of M`・**‹ › の
  セグメンテッドコントロール**(ツールチップ: Previous match / Next match。
  **内容サイズ・左寄せ — バー全幅に伸ばさない**)。
  次(›)= ⌘G / 前(‹)= ⇧⌘G のショートカット(Conversations ウィンドウがキーのときのみ)。
  端まで行ったら反対側へ折り返す(ラップ)
- **ヘッダ下の区切り線は常時**(検索バーの有無によらず)
- **running の会話でも下部パネルは常に出す**: Resume を disabled + 左に
  `This conversation is running` を淡く(§4.6)
- **⟳ の押下反応**: 実行中はボタン disabled + ステータス行 `Refreshing…`(§4.6)
- **サイドバー境界の手がかり**: ホバーでリサイズカーソル + 境界線を濃く(§8.38(8))
- **ヒットが 1 件でも、押下のたびに現在のヒットへスクロールする**(現在位置が変わらなくても
  再スクロール)。**畳みの中のヒットへの遷移が動かない報告あり** — 再現テストを書いてから直す
  (自動展開 → スクロール → ハイライト張り直しの順序を疑う)
- **サイドバー幅を境界ドラッグで可変に**: 初期 250pt・200〜400pt・幅を UserDefaults に記憶(§9)
- **初回描画が真っ白のままになることがある**(何か操作すると回復)— ページの ready 前に
  ペイロードが落ちる競合・読み込み中の会話選択の取り違えを疑う。再現手順の特定と、
  取りこぼしても必ず再送される仕組み(ready 後の再送・世代番号など)で直す

### T6: ミニマップ・下端の手がかり・読み味の一致(§4.6 / §8.39)

- **ミニマップ = スクロールバー**(右端・常時・幅 20pt — §9): 略図(帯 = 濃い縞 / 本文 = 淡い塊 /
  コード = 中間。文字は描かない)+ 半透明グレーのつまみ(ドラッグでスクロール・ホバーで濃く)+
  地のクリックでジャンプ。右ペイン内の OS スクロールバーは非表示。検索中はヒットの線
  (現在位置は濃く)を重ねる — 既存の目盛りオーバーレイはミニマップに置き換えて削除。
  選択・コピーには入れない
- **終端マーカー**: 最後のメッセージの下に罫線ではさんだ淡い一行 `Latest message · <経過時間> ago`
  (経過時間はヘッダと同じ整形)。**上端スクロールシャドウ**: 上に続きがある間だけヘッダ下に薄い影
- **タイポグラフィの一致**: 実装の行間・段落間・ブロック間隔・帯の余白を conversations-design.html
  の CSS 値と突き合わせて監査し、一致させる(「詰まって見える」の解消。目視ではなく値で合わせ、
  差分を完了報告に表で載せる)。**帯の直下と最初の返答の間隔**(近すぎると実機で指摘)は
  監査項目に必ず含める(html の `.ans` の上余白が正)。**Resume ボタン**はモックの .btn 相当の大きさに(controlSize を明示)
- **計装ログの整理**: 正常系の毎回のログ(load dispatch / rendered ack)を削除。異常系
  (held/flush・自己修復・再注入・プロセス死・JS エラー)は既定レベルのまま残す

### T7: 検証

- ペイロードの形(ブロック・セル・オフセットの UTF-16 変換)は Core 側でテストする
  (ゴールデンテスト可)。既存の 198 テストは全緑のまま
- `(cd app && swift build && swift test)` / `cargo test --workspace --locked` /
  `cargo clippy --all-targets --locked` 全緑 / 変更ファイルの CJK 掃引
- 悪意入力の確認を自動化する: `<script>` / `onerror` / `javascript:` リンクを含む会話を
  ページに流し、スクリプトが実行されず文字として出ることを検証する(スパイクの手法を踏襲)
- テストはマシンの外部状態に触れない(CLAUDE.md の一般則): ブラウザ起動などの外向きの動作は
  注入可能にしてスパイで検証、フィクスチャの URL は `https://example.invalid/` 系を使う
- 実機確認は所有者(スモーク項目: またぎ選択と ⏺・ボタンの非選択 / Copy・Copy as Markdown と
  ショートカット / 打鍵中に選択が消えないこと / 表の折り返し / インライン code の見た目・行間 /
  巨大会話の体感 / ライト・ダーク / VoiceOver でメッセージが読めること)
