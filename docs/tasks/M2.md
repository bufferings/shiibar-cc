# M2 実装指示書: shiibar-client + shiibarctl(list/watch/wait/focus/focused/remove/doctor)+ install.sh

> 読む順番: ルートの `CLAUDE.md` → この指示書 → 参照節の `docs/DESIGN.md`。
> 仕様の正は DESIGN.md。食い違いを見つけたら実装で解決せず、完了報告で指摘すること。
> M1 は完了済み(proto の型・codec・extract、shiibard 一式、`shiibarctl report`、hooks)。それらの API は流用する。

## ゴール

daemon に対する読み取り・待機・ジャンプの CLI を完成させ、日常投入(ドッグフーディング)を開始できる状態にする。
完了条件(DESIGN.md §6 M2):
- 自動テスト: exit code 系統(wait: 0/1/2/124、focus: 0/1/2/3)、selector 解決、iterm モジュールの純関数部
- 実機スモーク: `wait --status done && say done` が動く / focus で該当タブが前面に来る / doctor が全項目 green

## 成果物

### 1. `crates/shiibar-client` — クライアントライブラリ(DESIGN.md §4.3)

- **接続・リクエスト**: proto の Request/Response 型と codec を使い、socket に 1 リクエスト送って 1 レスポンスを読む
  ヘルパー。状態ディレクトリ解決は M1 の `shiibarctl` と同じ規則(`SHIIBAR_STATE_DIR` → 既定 `~/.local/state/shiibar/`)。
  proto かここに共通化してよい(M1 の shiibarctl 側と重複するなら整理する。ただし proto の既存 API は壊さない)
- **list / subscribe**: subscribe は接続を保持して SubscribeEvent を 1 件ずつ返すストリーム。
  最初に必ず snapshot が来る前提を利用する
- **wait(selector, status, timeout)**: subscribe 1 本で実装(§4.3。最初の snapshot で現状確認 → 条件一致まで待つ。
  別途 list を叩かない)。戻り値で「成立 / タイムアウト / 対象消滅(agent_removed)」を呼び出し元が区別できること
- **iterm モジュール**(iTerm2 の知識はここにのみ置く。DESIGN.md §4.3・§8.2〜。設計原則2):
  - **テスト分離が最重要**: 「AppleScript ソースの生成」と「osascript 出力のパース」を純関数として切り出し、
    osascript プロセス実行部だけを差し替え可能にする(trait かクロージャ注入)。純関数部を単体テストする
  - `focus(target)`: target の `:` 以降の UUID を取り出し(形式 `wNtNpN:UUID`)、AppleScript で
    iTerm2 の windows→tabs→sessions を走査して一致セッションの tab を select・window 前面化・activate。
    一致なし(`session:` フォールバック target 含む)は「該当なし」。
    **iTerm2 が未起動なら起動せずに「該当なし」を返す**(`activate` を走査成功後まで遅延する)
  - `focused()`: iTerm2 が最前面のとき前面 window の current session の target を返す。前面でなければ「なし」
  - `open_tab(cwd, cmd)`: 新規タブを cwd で開いてコマンド実行(resume 用。M3 で使う。M2 では未使用でも実装してよいが、
    スコープを絞るなら M3 に回して可。判断を完了報告に書く)
  - osascript の **TCC(オートメーション)権限エラーを「該当なし」と区別して**呼び出し元に返す(型で分ける)

### 2. `crates/shiibarctl` — 残りのサブコマンド(DESIGN.md §4.4)

`report`(実装済み)以外を実装する: `list [--json]` / `wait` / `watch` / `focus` / `focus -` / `focused` / `remove` / `doctor`。
`resume` は M3 なので**やらない**(スタブも作らない)。

- **selector**: target の完全一致、または `.`(カレントディレクトリと cwd が一致するエージェント)。
  cwd 部分一致は非スコープ(§8.10)。`.` で複数一致 or 不一致のときの扱いを決めて完了報告に書く
- **exit code(全サブコマンド共通、§4.4)**: 0 成功 / 1 接続・内部エラー(daemon 不在含む、stderr に理由) /
  2 該当なし(wait は対象消滅。理由は stderr) / 3 osascript 権限(TCC)エラー / 124 wait タイムアウト。
  `report` の「常に 0」は M1 実装済みで、この規則の唯一の例外
- **list**: `--json` は proto の list 応答をそのまま。非 `--json` は「状態 / ラベル / 経過時間 / target」の整列テキスト。
  ラベル整形(cwd をホーム配下なら `~` 起点で末尾 2 要素)は §4.5 と同じ規則。この整形は後で app と CLI 双方で
  使うので shiibar-client 側に置くのが望ましい
- **watch**: subscribe のイベントを行 JSON で標準出力へ流し続ける
- **focus / focus -**: §4.4。成功時に「移動前の前面 target」を `last_focus`(状態ディレクトリ内)に保存し、
  daemon に `seen` を送る。`focus -` は last_focus に戻り、自身も移動前 target を last_focus に保存(cd - と同じトグル)
- **doctor**: socket 疎通(`info`)/ daemon version・last_report_at / hooks 設定の有無
  (`~/.claude/settings.json` に report.sh を含むか)/ `shiibarctl` が PATH にあるか / osascript TCC 権限
  (無害な iTerm2 走査を 1 回試す)を順に検査して人間向けに報告

### 3. `scripts/install.sh` / `uninstall.sh`(DESIGN.md §5、§4.5 の同梱方針)

- M2 段階のスコープ: **バイナリ配置 + hooks 案内**。`.app` 化・Login Items・CLI symlink の .app 由来部分は M4。
  - `cargo build --release` した `shiibard` / `shiibarctl` を配置(例: `~/.local/bin/`)
  - `hooks/report.sh` を配置し、`hooks/settings-snippet.json` を `~/.claude/settings.json` にどうマージするか案内
    (自動マージまでやるか案内に留めるかは判断して完了報告に書く。既存 settings を壊さないこと)
  - `uninstall.sh` は配置物の削除と、settings から hooks を外す案内
- ドッグフーディング用の `scripts/dev-reload.sh`(再ビルド → 手動 daemon 差し替え)は任意。作るなら簡潔に

## テスト要件(受け入れ条件)

- **exit code の全系統**: wait(0/1/2/124)・focus(0/1/2/3)・その他(daemon 不在=1)を検証。
  focus の 2(該当なし)と 3(TCC エラー)は、osascript 実行部を注入で差し替えて模擬する
  (実機 osascript に依存しないテストにすること)
- **iterm 純関数部**: AppleScript 生成(target→UUID 抽出、`session:` フォールバックの扱い)と
  osascript 出力パースの単体テスト。実際の osascript は呼ばない
- **selector 解決**: 完全一致 / `.`(cwd 一致)/ 不一致・複数一致の分岐
- **wait**: 既存 blocked に即成立 / 出現待ち→成立 / タイムアウト(124)/ 対象消滅(2)。
  daemon は M1 の実バイナリ or ライブラリを一時 `SHIIBAR_STATE_DIR` で起動して結合テスト
- すべてのテストは一時状態ディレクトリを使う(実ディレクトリに触れたら不合格)
- `cargo test` 全緑、`cargo clippy --all-targets` 警告ゼロ
