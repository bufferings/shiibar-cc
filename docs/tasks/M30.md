## タスク

### T1: Keep on Top トグル

- アプリメニューに **Keep on Top** を追加。位置は最終グループの先頭(Close Window の上)。
  チェックマークで現在値を示す。ショートカットは付けない
- ON の間、Agents ウィンドウのレベルを floating に(OFF で通常レベルへ)。
  **変えるのはレベルだけ** — collectionBehavior(Space 追従)には触らない
- 値は UserDefaults に永続(既定 OFF。既存の設定キー命名に倣う)。
  切り替えは表示中のウィンドウに即反映。次にウィンドウを開いたときは記憶値を適用
  (適用タイミングは既存の配置機構(AgentsWindowPlacer / AgentsWindowViewModel)の
  開き検知に乗せる — 新しい監視機構を作らない)
- メニューのチェック状態は `AppMenuModel`(重複排除ファサード)経由で観測する
  (AppState を直接観測してメニューを churn で殺した M27 の教訓 — 値の変化時だけ再構築される形に)

### T2: 検証

- `(cd app && swift build && swift test)` 全緑。
  `cargo test --workspace --locked` / `cargo clippy --all-targets --locked` 全緑(Rust 無変更)
- view なしで検証できるロジック(設定の永続・既定値等)は Core に切り出してテスト
  (無理に mock 機構を新設しない — 不可能なら理由を報告)
- 実機確認は所有者(スモーク項目: メニューの ON/OFF とチェックマーク / ON で iTerm2 の上に
  浮く・クリックで行に飛べる / OFF で通常の重なりに戻る / ON のまま閉じて開き直すと
  最前面のまま / 再起動後も記憶値が効く / エージェント churn 中にメニューが閉じない(既存挙動))
