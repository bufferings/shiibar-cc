## 背景とゴール

ドッグフーディングで見つかった小さな棘 2 つをまとめて抜く:

1. 完了通知のバナーに `last_assistant_message` の生マークダウン(`**`・バッククォート等)が
   そのまま表示される(実観測: `All four links still return **`200`**`)
2. 新規インストール直後、アプリのアイコンが汎用のままになることがある
   (DEVELOPMENT.md「アイコンまわりの開発メモ」5 項。`lsregister -f` と `killall Dock` の
   どちらが効いたかは未特定だが、自動化できて無害な `lsregister -f` を install.sh に入れる)

## タスク

### T1: `last_assistant_message` のマークダウン除去(`crates/shiibar-cc-proto/src/extract.rs`)

- DESIGN.md §4.1 の仕様どおり、**切り詰め(200 文字)の前に**除去する:
  - バッククォート文字を除去(囲まれた内容は残す)
  - `**` と `__` のペアを除去(内容は残す。閉じが無い単独出現は触らない)
  - 行頭(および行頭の空白後)の `#` 連続 + 空白の見出しマーカーを除去
  - `[text](url)` → `text`
  - **単独の `*` / `_` は除去しない**(snake_case・パス・強調でない用例への誤爆を避ける)
- 純関数として実装し、単体テストを書く(少なくとも: 実観測例
  `All four links still return **`200`** — no broken links` / 見出し付き複数行 / リンク /

- 新しい外部クレートを追加しない

  `lsregister` のフルパスは DEVELOPMENT.md に記録のある
  `/System/Library/Frameworks/CoreServices.framework/Frameworks/LaunchServices.framework/Support/lsregister`
  を使い、存在しない場合は黙ってスキップ(インストールを止めない)
- `docs/DEVELOPMENT.md` の「アイコンが反映されないとき」5 項を現状に合わせて更新する
  (install.sh が `lsregister -f` を自動で行うようになったこと。`killall Dock` は手動の
  フォールバックとして残す)
