## 背景とゴール

所有者要望: Setup Check ウィンドウの見た目を、M14 の Settings ウィンドウと同じ
SwiftUI `Form` + `.formStyle(.grouped)`(System Settings 風の囲みカード)に揃える。

**view 層のみの変更**。チェックの実行と判定
(`SetupCheckLogic` / `SetupCheckParsing` / `CLIRunner` / `NotificationManager` /
doctor が正の原則)には一切触らない。

## タスク

### T1: `app/Sources/ShiibarCcApp/SetupCheckView.swift` の view 置き換え

- 現行の VStack + Divider + ScrollView 構成を `Form` + `.formStyle(.grouped)` に置き換える
- **2 セクション**: doctor 由来の行(セクション見出し **"Doctor"**)と、アプリ側 2 行 =
  通知許可・Start at Login(セクション見出し **"App"**)。
  ViewModel は現在結合済みの単一 `rows` を公開しているので、`cliRows` / `appRows` の
  2 つの published 配列に分けて公開する(結合順・内容は不変)。`runChecks()` の
  実行フロー — 多重実行ガード・バックグラウンド実行・メインアクターへの hop — は変えない
- **内側の "Setup Check" 見出し行は削除**(タイトルバーが兼ねる。Settings ウィンドウと
  同様、ウィンドウ内に大見出しは置かない)
- **Re-run ボタンは Form の外、下部の常時表示フッター(trailing)へ移す**
  (スクロールに巻き込まない)。実行中は ProgressView + disabled という現行挙動のまま
- 各行の中身は現行のまま: ✓/⚠/✗ シンボル(緑/黄/赤)+ summary + 薄字 hint。
  `SetupCheckRowView` を Form の行として使い続けてよい
- 空状態(結果ゼロかつ非実行中)の "No results yet." 表示は維持(placement は Form 内でよい)
- `onAppear` の `NSApp.activate` + 自動 `runChecks()` は変えない
- ウィンドウ寸法は grouped Form の余白に合わせて調整してよい(目安: 現行 420×360 から
  大きく外れない)。行数は環境依存なのでリストはスクロール可能なままにする
