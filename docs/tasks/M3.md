## 成果物

### 1. `crates/shiibar-cc-client` — iterm モジュールに `open_tab(cwd, cmd)` を追加(DESIGN.md §4.3)

- 新規 iTerm2 タブを `cwd` で開き、`cmd` を実行する。**focus と違い、iTerm2 が起動していなければ起動してよい**
- 既存パターンを踏襲する: AppleScript ソースの生成と osascript 出力のパースは純関数として切り出し、
  実行部は `AppleScriptRunner` 差し替えで単体テスト可能にする
- TCC(オートメーション)権限エラーは「失敗」と区別して呼び出し元に返す(既存の focus と同じ)
- `cwd` / `cmd` は AppleScript 文字列に埋め込むので、引用符・バックスラッシュのエスケープを純関数として
  実装しテストすること(パスに `"` や `\` を含むケース)

### 2. `crates/shiibar-cc` — `resume` サブコマンド(DESIGN.md §4.4)

処理の流れ:

1. daemon に `{"cmd":"sessions"}` → 履歴(last_seen 降順)を得る
2. daemon に `{"cmd":"list"}` → **実行中セッション(list に載っている `session_id`)を候補から除外**する
   (`claude --resume` の二重起動防止)
3. 選択 UI: **fzf が PATH にあれば fzf、なければ番号選択のプロンプト**。
   候補行は人間が判別できる形式(cwd の短縮ラベル・last_status・経過時間・session_id を含む)にし、
   選択結果から session_id を一意に復元できること
4. 履歴の `cwd` がディスク上に存在しなければ **stderr に警告して `$HOME` で開く**
5. `open_tab(cwd, "claude --resume <session_id>")` を実行する

- exit code(§4.4 共通規約): 0 成功 / 1 接続・内部エラー / **2 該当なし = 候補ゼロ(履歴なし・全員実行中)
  および選択の中断(fzf abort / プロンプトの EOF)**(理由は stderr)/ 3 osascript の TCC エラー
- 選択 UI も「候補整形・選択結果の解釈」を純関数に切り出し、fzf / プロンプトのプロセス実行部だけを分離する
  (fzf の有無に依存しないテストを書くため)

  1000 行超で書き直し)を覆っているか確認し、不足があれば追加する
- 実行中セッションの除外: list の session_id 集合との突き合わせを純関数でテスト
  (除外で候補ゼロになるケース = exit 2 も含む)

- テストは一時ディレクトリの state dir のみ使用(実 `~/.local/state/shiibar-cc/` に触れない)。
  マシン固有の絶対パスを書かない(ルート CLAUDE.md)
