## 背景とゴール

所有者の実運用フィードバック: 毎日見続ける一覧ウィンドウの中に、低頻度の操作(⌄)が常駐して
場所を取るのはおかしい。操作は「窓がある間だけ通常アプリになり、画面上端のアプリメニューに出す」
方式に決定(§8.30)。あわせて、常時表示する窓のための外観設定(System / Light / Dark)を加える。

## タスク

### T1: 窓がある間だけ通常アプリ化

- Agents 窓が開いたら `NSApp.setActivationPolicy(.regular)`、閉じたら `.accessory` に戻す。
  開閉検知は `AgentsWindowViewModel` の既存機構(didBecomeKey + willClose のタイトルフィルタ)に乗せる
- 切り替え直後にメニューバーの表示が更新されない AppKit の既知の癖に注意 —
  policy 変更と `NSApp.activate` の順序を実機で確かめて確実な順に固定し、報告に記録する
- Dock アイコンのクリック・⌘Tab での復帰で窓を前面に出す
  (`applicationShouldHandleReopen` 等。通常アプリでいる間しか起きないので、対象は常に Agents 窓)
- Settings / Setup Check を単独で開いても切り替えは起きないこと(条件は Agents 窓のみ)
- アプリ起動時・Quit 時の既存フロー(accessory 設定・daemon 停止)は変えない

### T2: アプリメニュー

- 通常アプリ時のメニューは**アプリメニュー 1 個だけ**。File / Edit / Window メニューは出さない
  (SwiftUI の `.commands` で標準グループを除去する)。構成と順序は DESIGN.md §4.5 の
  「アプリメニューは 1 個だけ」の項のとおり:
  About / ─ / Settings…(⌘,)/ Setup Check… / ─ / Rescan(⌘R)/ Clear badges / Sort by(サブメニュー)/
  ─ / Close Window(⌘W)/ Quit Shiibar CC(⌘Q)
- 各項目は既存の実装(VMenuHandler / AppState)と同じ動作につなぐ:
  - Clear badges は未読ゼロで disabled(⌄ と同じ条件)
  - Sort by はラジオ 3 択でチェックマークが現在値を映す。変更は両方の器に即反映(既存挙動)
  - Close Window は key ウィンドウを閉じる標準動作
  - Quit は `state.quit()`(daemon の shutdown ack を待つ経路)。素の `terminate` に直結しない
- メニューから Settings… / Setup Check… を開いても Agents 窓は閉じない(そもそも閉じる理由がない)

### T3: 窓から ⌄ を撤去 + Rescan のステータス行

- 窓コンテナでは topbar(⌄ チップの行)を描画しない。窓の中身 = 一覧 + 警告行のみ
- メニューから Rescan したとき、窓が表示中なら**窓下端のステータス行**(警告行と同じ場所)に
  一時表示を出す: Rescanning… / ✓ Rescan done / Rescan failed。文言・表示時間(§9 の
  「Rescan 結果の一時表示」= 2 秒)は既存の `RescanFeedback` を再利用する。
  ドロップダウン側の表示(⌄ の隣)は従来どおり
- M26 で入れた器分岐のうち不要になるもの(窓側の ⌄ 表示・`showsOpenAsWindowMenuItem` 等)は
  削除して単純化してよい(ドロップダウンの挙動は 1px も変えない)

### T4: ドロップダウン ⌄ の並び順をアプリメニューと統一

- 新しい並び(DESIGN.md §4.5): About / ─ / Settings… / Setup Check… / ─ /
  Rescan / Clear badges / Sort by / ─ / Open as Window / Quit
  (Open as Window はアプリメニューの Close Window に当たる位置 — 最終グループは
  「窓の動詞 + Quit」で、窓の動詞だけが器で入れ替わる)
- 既存の挙動(全項目クリックで閉じる・Clear badges の disabled・ラジオのチェック・
  popUp 後の makeKey)はそのまま維持する

### T5: Appearance 設定(System / Light / Dark)

- Settings ウィンドウの General グループに **Appearance** ポップアップを追加
  (Start at Login の下)。選択肢は System / Light / Dark、既定 System
- 選んだ瞬間に `NSApp.appearance` を切り替える(System = nil / Light = aqua / Dark = darkAqua)。
  UserDefaults に永続し、アプリ起動時に再適用する
- 文字列 ↔ 外観の対応など view なしで検証できるロジックは Core に切り出してテストを足す
  (既存の注入パターンに倣う。無理に mock 機構を新設しない)

### T6: 検証

- `(cd app && swift build && swift test)` 全緑。
  `cargo test --workspace --locked` / `cargo clippy --all-targets --locked` 全緑(Rust 無変更の確認)
- 実機確認は所有者(スモーク項目: 窓を開くと Dock・メニューが現れ、閉じると消える /
  Dock クリック・⌘Tab で窓が前面に出る / メニューの全項目とショートカット(⌘, ⌘R ⌘W ⌘Q)/
  窓に ⌄ が無い / メニューから Rescan で窓下端に一時表示 / ドロップダウン ⌄ の新しい並び /
  Appearance の 3 択が即時に効き、再起動後も保持される / ダーク外観で窓・ドロップダウンが
  読みやすい / ドロップダウンのその他の挙動が従来どおり)
