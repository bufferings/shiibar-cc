## 背景とゴール

所有者の実運用フィードバック: 一覧を「ずっと表示しておきたい」。ドロップダウンはよそを
クリックすると閉じるので、複数エージェントを並行で回している日には向かない。

ゴール: ⌄ メニューに **Open as Window** を追加し、ドロップダウンと同じ一覧を
**普通のウィンドウ**(タイトル `Agents`)として出しっぱなしにできるようにする。
仕様の全文は DESIGN.md §4.5 の「一覧ウィンドウ」項。要点:

- タイトルバー非表示(`hiddenTitleBar`。信号機のみ)の普通のウィンドウ: 最前面固定なし・
  全 Space 追従なし・外側クリックで閉じない・リサイズ不可
- 位置は毎回「Open as Window を押した時点のドロップダウンパネルと同じ位置」(位置記憶なし。
  既に開いていたら前面に出して同じ位置へ戻す)
- 行クリックで focus してもウィンドウは閉じない
- スピナーは表示中だけ回す。経過時間の基準とフラット 2 モードの並び順は、開いた時点 +
  表示中毎分の取り直し。通知許可の denied 再評価も開くたび
- ドロップダウンは従来どおり残す(挙動を一切変えない)

## タスク

### T1: 一覧ビューの共有化(リファクタ)

`DropdownView` の中身(TopBar / 一覧(grouped・flat)/ No agents / 警告行)を、ドロップダウンと
一覧ウィンドウの両方から使える形に切り出す。制約:

- **ドロップダウンの挙動は 1px も変えない**(経過時間の固定、`flatOrderSnapshot`、
  `isDropdownOpen` によるスピナー・ホバーのゲート、行クリックで閉じる、すべて従来どおり)
- 器ごとに違う点はコンテキスト(例: enum か設定構造体)で注入する:
  - 行クリック: dropdown = focus + `dismissDropdown()` / window = focus のみ
  - スピナー・ホバーのゲート: dropdown = `isDropdownOpen` / window = ウィンドウ可視フラグ(T3)
  - 経過時間・フラット並び順の基準: dropdown = `dropdownOpenedAt` + `flatOrderSnapshot` /
    window = T3 の窓用基準
  - ⌄ メニュー: window では **Open as Window 項目を出さない**。また window 内では
    `dismissDropdown()` を呼ばない(Settings… / Setup Check… を開いても窓は閉じない)。
    それ以外の項目(Rescan / Clear badges / Sort by / Settings… / Setup Check… / About / Quit)は
    両方で同一に動く。Sort by の変更は両方の器に(それぞれの流儀で)映る

### T2: Window シーンと Open as Window 動線

- `ShiibarCcMenuBarApp` に `Window("Agents", id: ...)` シーンを追加(Settings / Setup Check と
  同じパターン。`windowResizability(.contentSize)`。ドロップダウンと同じ幅 340・一覧の
  maxHeight 360 スクロールが器越しにそのまま成立すること)
- ⌄ メニュー(dropdown 側のみ)に **Open as Window** を追加。位置は「表示と設定」グループの
  先頭(Sort by の上)。ハンドラは既存の `openSettings` パターン + 位置合わせ:
  1. ドロップダウンパネル(anchor の `window`)の**画面フレームを先に取得**
  2. `dismissDropdown()` / `NSApp.activate(ignoringOtherApps: true)` / `openWindow(id:)`
  3. 開いた `NSWindow` を特定して `setFrameTopLeftPoint` でパネルの top-left に合わせる
     (`openWindow` は即時に NSWindow を返さないので、特定は Settings / Setup Check で実績のある
     タイトルフィルタ方式(`SetupCheckViewModel.observeWindowLifecycle` 参照)か、
     main queue の次ターンで `NSApp.windows` から引く。**SwiftUI の frame 復元と戦わない** —
     復元がどう働こうと、Open as Window のたびに明示的に位置を上書きする、が仕様)
- 既に開いている場合: 前面に出して同じ位置へ戻す(新規に開いた場合と同じ 3 手順で成立するはず。
  `openWindow(id:)` は既存ウィンドウを前面化する)

### T3: 窓の生存状態(可視フラグ・毎分tick・許可再評価)

- ウィンドウ可視フラグ: `NSWindow` の通知をタイトルでフィルタして追う
  (`SetupCheckViewModel.observeWindowLifecycle`(M16)と同じ流儀。ドロップダウンの
  `observeDropdownOpen` のクラス名フィルタとは別物なので混ぜない)
- 窓用の経過時間基準と flat 並び順スナップショット: 窓が**開いた時点で取り直し**、
  **可視の間は 60 秒ごとに取り直す**。タイマーは可視の間だけ動かす(不可視で止める —
  トレイの「タイマーは working 表示のときだけ」と同じ倹約)。ドロップダウン側の
  `dropdownOpenedAt` / `flatOrderSnapshot` には触らない(共有しない — 両方開いたときに
  互いを乱さないため)
- 窓が開いたとき `notificationManager.refreshPermissionStatus()` を呼ぶ(M25 と同じ趣旨)

### T4: タイトルバーの非表示化(器の見た目)

- Window シーンに `.windowStyle(.hiddenTitleBar)` を適用。信号機ボタンは残す(閉じる動線は
  赤ボタンと ⌘W のまま)
- ウィンドウの `title` は `Agents` のまま保持する(T3 のタイトルフィルタ・T2 のウィンドウ特定・
  Mission Control 表示が依存)。タイトルフィルタと位置合わせが引き続き機能することを確認する
- 一覧のレイアウトは窓用に調整しない: 一覧(topbar の ⌄ 含む)は信号機の帯の**下**に、
  ドロップダウンと同一の配置で置く(safe area の打ち消しや窓専用の余白を入れない)。
  ドロップダウン側のレイアウトも 1px も変えない
- 信号機の帯を掴んでウィンドウをドラッグできること(SwiftUI が旧タイトルバー領域の
  ドラッグを維持するならそれでよい。効かない場合は最小の手当てを、内容を報告した上で行う)

### T5: 検証

- `(cd app && swift build && swift test)` 全緑。切り出したロジックのうちビューなしで
  テスト可能なもの(器コンテキストの分岐・毎分 tick の基準取り直し等)は既存の注入パターンに
  倣ってテストを足す(無理に mock 機構を新設しない — 不可能なら理由を報告)
- `cargo test --workspace --locked` / `cargo clippy --all-targets --locked` 全緑(Rust 無変更の確認)
- 実機確認は所有者(スモーク項目: 開く位置 / 外側クリックで閉じない / 行クリックで focus しても
  残る / スピナー / 毎分の経過時間更新 / 再 Open as Window で位置が戻る / ⌘W で閉じる /
  タイトルバーが出ない・一覧の配置がドロップダウンと同じ・信号機の帯ドラッグで動く /
  ドロップダウンが従来どおりであること)
