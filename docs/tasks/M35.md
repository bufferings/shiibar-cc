## 背景

M34 の CLI(`conversations index` / `search` / `show`)の上に Conversations ウィンドウ(§4.6)を組む。
アプリは **CLI を呼んで JSON を読んで描くだけ** — Swift に transcript パース・SQLite・生死判定は
持ち込まない。合わせて最低 OS の引き上げと README を仕上げる。M33〜M35 の完成で v0.4.0 として
リリースする(リリース作業自体は所有者)。

## タスク

### T1: 最低 OS を 14 に引き上げ

- `app/Package.swift` の `.macOS(.v13)` → `.macOS(.v14)`(冒頭コメントの `macOS 13+` も 14+ に)
- `scripts/lib/bundle.sh` が生成する Info.plist の `LSMinimumSystemVersion`(`13.0` → `14.0`)
- `packaging/homebrew/` の cask テンプレートの `depends_on macos:`(`:ventura` → `:sonoma`)
- 理由は §8.34(Ventura の更新終了 + 同梱 SQLite の trigram 保証)

### T2: subprocess ランナー

- 既存の `CLIRunner` は同期・全バッファ読みで、この画面には使えない。**非同期・行ストリーム読み・
  キャンセル可能なランナーを新設**する: stdout のパイプから行を逐次読める(index の進捗用)/
  実行中のプロセスを terminate できる(検索のキャンセル用 — 追いつき書き込み中に SIGTERM しても
  SQLite の規律 — §4.6 — で安全)/ 完了時に exit code と全出力を返せる
- Resume(T6)は従来どおり既存機構でよい

### T3: ウィンドウ

- 2 ペイン(左 = 検索欄 + ⟳ + 一覧 + ステータス行、右 = 会話全文 + Resume)。見た目・状態の細部は
  `docs/conversations-design.html` が正
- クロームは一覧ウィンドウと同じ(タイトルバー非表示 + 信号機の帯。`title` は `Conversations`)。
  初期 640×480pt(左ペイン 280pt)・縦横リサイズ可・**サイズと位置を記憶**(§9。UserDefaults 永続 —
  アプリ再起動を跨ぐ。Agents 窓の高さ記憶と同じ)。閉じるのは赤ボタンと ⌘W
- **窓が存在する間だけ通常アプリになる規則(§8.30)の条件を「一覧ウィンドウまたは Conversations
  ウィンドウ」に拡張**(§4.5)。Keep on Top は一覧ウィンドウ専用のまま — 一覧ウィンドウが無い間は disabled

### T4: 一覧・検索・ステータス行(左ペイン)

- データは `conversations search --json`(§4.4 の 5 フィールド)。**全件表示**(live も出す)。
  行 = タイトル(null はフォルダラベルに落とす)/ フォルダラベル(§4.5 の末尾 2 要素規則を cwd から
  整形)+ 経過時間。**live の行は 2 行目に `running` の淡いテキスト印**。状態記号は使わない
- 検索欄: インクリメンタル(デバウンス 200ms — §9 — で search 実行、進行中はキャンセル)。
  **trim 後に有効な語(2 文字以上)が 1 つも無い間は発行せず** browse 一覧のまま。プレースホルダ
  `Search (2+ characters)`。**⟳ ボタン**(検索欄の隣): 同じクエリで search を再実行。
  **search が exit 1 のときは前回の結果を維持**してステータス行にエラー(§4.6)。
  検索が全構築に当たって長引く場合は `index --json` を起動して進捗をステータス行に出してよい
  (flock と中継 — §4.4 — が安全性を担保する)
- **表示更新の契機は 4 つだけ**(§4.6): 開いたとき / 打鍵 / ⟳ / Resume 成功直後。それ以外で
  勝手に更新しない
- ステータス行: `N conversations (M running)` / 絞り込み中 `N of M conversations` / 構築中は進捗 /
  エラー時: **index のエラーは一覧を出さずにエラー表示、search のエラーは前回結果を維持して
  エラー表示**(§4.6)。空状態は `No conversations`、検索 0 件は `No matching conversations`
  (conversations-design.html)。絞り込み中の分母 M は直近の全件数(空クエリ結果または直近の
  search 結果の総数)でよい
- 一覧の差分描画と選択保持は session_id をキーにする(更新で行がちらつかない・選択が飛ばない)

### T5: 会話の表示(右ペイン)

- 行を選んだら `conversations show --json` で**全文を 1 回で取得**し、古い順・下端 = 最新で表示
  (遅延描画 — 最大級で 280KB。§7-6。展開の先読みはしない)。
  **show が exit 2(索引に無い)ならプレビューをクリアしてステータス行に一言、exit 1 なら
  エラー表示(一覧は維持)**(§4.6)
- **ヒットナビゲーション**: show 取得後、**各語の全出現**をアプリが手元で計算し
  (大文字小文字無視の部分一致 — CLI と同じ意味論)、**本文に 1 件以上あるときだけ**検索バー行
  (文書順カウンタ + ▲ = 古い方へ / ▼ = 新しい方へ)を出す。全語を**同色でハイライト**し、
  現在位置のみ濃く。**ハイライトは背景色属性で行い、太字化しない**(字幅が変わって折り返しが
  動くと「スクロール不動」の規則が壊れる)。初期位置 = どれかの語の**最新の出現**。0 件なら
  バーを出さず下端から。クエリ打ち替えでハイライトとカウンタは再計算、スクロールは動かさない
- **状態規則**(§4.6): 一覧の更新はプレビューに触らない(選択中の会話が結果から消えたときだけ
  クリア)。**スクロール位置は会話ごとに記憶**(ウィンドウを閉じたら破棄。復帰時は記憶が初期位置に
  勝つ)。記憶の粒度は**メッセージ単位で足りる**(`scrollPosition(id:)` — macOS 14。ピクセル精度は
  求めない)。**500 文字超のメッセージは畳んで `Show full message` で展開**(畳み内のヒットも
  カウンタに数え、▲▼ ジャンプで自動展開)
- live の会話も同様に表示できる(内容は最後の索引時点まで)

### T6: Resume(past の行のみ)

- `shiibar-cc resume --cwd <cwd> <session-id>` を subprocess 実行(focus と同じ呼び出し機構)。
  ウィンドウは閉じない。exit 3(TCC)は focus 失敗時の扱いに合わせて伝える。exit 1 は
  ステータス行にエラー表示
- **成功直後に search を再実行**(§4.6 — 再開した会話が running 表示に変わる)

### T7: 索引の温めとウィンドウを開いたときの流れ

- **アプリ起動時**に `conversations index` をキック(失敗は静か — ログのみ)。
  `NSBackgroundActivityScheduler` で**約 10 分ごと**にも実行(§9。定期 reconcile — §8.22 — の実装の
  流儀に合わせる)。**FSEvents は使わない**(§4.6)
- **ウィンドウを開いたとき**: `conversations index --json` を実行し、進捗の行 JSON を読んで
  ステータス行に `Indexing N of M…` を表示。**全構築中は検索欄を無効**にし、完了後に search で
  一覧を出す(温まっていれば index は即 done → 体感なし)。二重起動の管理は不要 — 排他と進捗中継は
  CLI 側の仕組み(§4.4)が担うので、アプリはいつ index を呼んでも同じ形の進捗が得られる

### T8: 入り口(メニュー)

- ⌄ メニュー: 最終グループの Open as Window の下に `Conversations…`(セパレーター追加なし。§4.5)。
  アプリメニュー: Close Window の下・Quit の上(§4.5)。どちらも**窓が存在する間は disabled**
  (Open as Window と同じ規約)。クリックでドロップダウンは閉じる(既存項目と同じ)
