## 背景

実データで Claude が表を多用し、生のパイプ記号は壊れて見える(§8.37)。GitHub 形式のパイプ表を
ブロック種別に加え、グリッドで描画する。アプリ(Swift)のみの変更。

## タスク

### T1: ブロック分解に表を追加(Core)

- **判定**: パイプを含む行(ヘッダ行)の直後に区切り行(`|`・`-`・`:`・空白のみで `-` を含む)が
  続いたら表。データ行は表の形をした行が続く限り。区切り行が無ければ表とみなさず段落のまま(§4.6)
- **モデル**: `MessageBlock` に表の種別を追加(セルは行 × 列の `AttributedString` — セル内インラインは
  既存の `inlineAttributed` を通す)。行頭・行末のパイプと区切り行は消費する。
  列数不一致など壊れた表は読めた形で出す(パースに失敗したら段落フォールバック — 落ちない)
- **描画後テキスト**: `|` 記号・区切り行・整形空白は含めない — **セルの見える文字だけ**(§4.6)。
  セル間・行間の区切りが 1 文字(既存のブロック結合と同じ流儀)になるよう joined 形を定義し、
  ヒットオフセットがセル単位で引けること(ブロック内のセル開始オフセット表)。
  検索語は空白を含まないので、ヒットがセル境界をまたぐことはない
- **畳み境界**: 既存の文字数計算(§9: 描画後テキスト 500 文字)にそのまま参加する。
  畳み位置が表の途中に落ちた場合は、そこまでのセルだけ見せる(境界をまたぐセルは prefix —
  既存のブロック切り詰めと同じ規則)

### T2: グリッド描画(app)

- ヘッダ行 = セミボールド + 淡い背景、行間に罫線、全列左揃え(列揃え指定 `:---:` は無視)。
  枠は角丸 + 罫線(conversations-design.html の `.tbl` が目安)
- 表は内容幅のまま(本文幅いっぱいに伸ばさない)。広い表は表の中だけ横スクロール
  (コードブロックと同じ文法)。読む幅の上限(約 60em)の内側に置く
- セルの文字サイズは本文と同じ(⌘ズームに追従)。セル内のヒットハイライトは既存の
  背景色注入と同じ規則(現在位置のみ濃く)

### T3: 検証

- 新ロジック(表の判定・セル分解・描画後テキスト・セルオフセット・畳みとの交差)は
  `ShiibarCcCore` に置いてテストを足す。壊れた表(列数不一致・区切り行なし・空セル)の
  ケースを含めること
- `(cd app && swift build && swift test)` 全緑 / `cargo test --workspace --locked` /
  `cargo clippy --all-targets --locked` 全緑(Rust 無変更)/ 変更ファイルの CJK 掃引
- 実機確認は所有者(スモーク項目: 表のグリッド表示 / 日本語セル / セル内コード・強調 /
  広い表の横スクロール / 表の中のヒットのハイライトと ▲▼ / 区切り行・パイプが検索に
  ヒットしないこと / ライト・ダーク)
