## 背景とゴール

実機観測(2026-07-09、brew 版 0.1.0): 通知許可を**アプリ起動後に**付けた場合、バナーは届くのに
ドロップダウンの警告行「Notifications permission denied」が出続け、アプリを再起動するまで消えない。

原因は特定済み: 警告行が見る `NotificationManager.permissionDenied` は、起動時の
`requestAuthorizationIfNeeded()`(`AppState.swift`)のコールバックで**一度だけ**設定され、
以後どこからも更新されない(`refreshPermissionStatus()` の呼び出し元がそこしかない)。
対照的に Setup Check の行は開くたびに `getNotificationSettings` を取り直すので正しい値を出す。

ゴール: **ドロップダウンを開くたびに許可状態を取り直し**、警告行が「開いた瞬間の真実」を映すようにする。

## タスク

### T1: 開くたびの再評価

- ドロップダウンが開いたタイミングで `notificationManager.refreshPermissionStatus()` を呼ぶ。
  開き検知は `AppState.observeDropdownOpen`(既存 — ドロップダウンの開閉を既に監視している)に
  乗せるのが最小。新しい監視機構を作らない
- 制約:
  - `getNotificationSettings` は非同期なので、開いた直後の一瞬は前回値が見え、取得後に
    警告行が消える/現れる可能性がある。この「開いている間の遅れた更新」は許容する
    (開くたびに最新へ収束することが要件。チラつき防止の凝った仕組みは作らない)
  - 起動時の `requestAuthorizationIfNeeded()` の流れは変えない
  - Setup Check には触らない(既に開くたび再評価で正しい)
