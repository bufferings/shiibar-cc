## T1. リネーム: Swift 内部名 / .app 名 / 表示名

単独コミットにする(機械的な差分を他と混ぜない)。

- Swift の `ShiibarCC` プレフィックスを `ShiibarCc` へ一括変更: Package.swift のターゲット
  (`ShiibarCcApp` / `ShiibarCcCore` / `ShiibarCcCoreTests`)、同名ディレクトリ 3 つ、型名・参照の全て
- 現在の `ShiibarCCAppApp` のような「App」二重の型名はこの機に整理する。モジュール名と同名の型は
  避けること(例: `ShiibarCcMenuBarApp`)
- `scripts/install.sh`: `APP_NAME="ShiibarCC.app"`、`CFBundleName` / `CFBundleDisplayName` は
  `Shiibar CC`、`CFBundleExecutable` は新実行ファイル名。**bundle ID(`cc.shiibar.menubar`)と署名
  identifier は変えない**(通知・Automation 許可を維持するため)
- install.sh は置き換え時に旧 `shiibar-cc.app`(既定パスのもの)を削除する。`scripts/uninstall.sh`・
  dev-reload スクリプト・`docs/DEVELOPMENT.md` のパス/名称参照も新名称に更新
- 完了報告に「再インストール(install.sh 再実行)が必要」と明記

## T2. ドロップダウンの反応(Rescan フィードバック + ホバー/プレス)

- **Rescan**(`AppState.runReconcile`, `app/Sources/.../AppState.swift:230` 付近):
  実行中は「Rescanning…」、正常終了後は「✓ Rescan done」を約 2 秒表示して消す(§9 に定数)。
  件数は出さない。exit 3(TCC)は既存の警告行の仕組みのまま。その他の非ゼロは一時表示
  「Rescan failed」を出す
- **ホバー/プレス**: セッション行(`RowView`)と `⌄` ボタンに `.onHover` の角丸ハイライトと
  プレス時のひと段階濃い表現を付ける(⌄ の中身 = ネイティブメニュー項目は OS 標準のまま)。
  色・角丸の値は `docs/menubar-design.html` が正(⌄ ボタンの扱いは T9 の項も参照)
- 判定ロジック(一時表示のステート遷移)は純ロジックとして単体テスト対象

## T3. ログイン時起動のトグル

- `⌄` メニューに「Start at Login」チェックボックスを追加。表示状態は毎回
  `SMAppService.mainApp.status == .enabled` を読む(自前フラグを状態の正にしない)。
  ON/OFF で register / unregister
- **起動時の無条件 register は廃止**。代わりに「初回のみ自動登録」: 自動登録を実施済みかだけを
  UserDefaults(`cc.shiibar.didAutoRegisterLoginItem`)に記録し、未実施ならバンドル内起動時に
  register して記録する。以後は二度と自動登録しない(OFF にした選択を上書きしないため)
- `--unregister-login-item` フラグ(uninstall.sh が使用)は維持
- 分岐ロジック(初回判定)は単体テスト対象

## T4. 通知の強化(抑制撤廃・掃除・本文・完了音)

**daemon / proto 側**:

- Stop フックの `last_assistant_message`(実 fixtures `stop_no_background_tasks.json` 等に存在)を
  `ReportPayload` に追加で受け、Stop→idle 遷移時にエントリへ保存(切り詰めは §4.1 どおり
  `shiibar-cc report` 側で先頭 200 文字、§9 に定数)。
  working への遷移で消す(`message` が waiting を離れると消えるのと同じ扱い)。ワイヤの `Agent` にも
  前方互換の追加フィールドとして載せる(§4.2)。遷移テーブル(§3.4)の状態・フラグ遷移自体は
  **一切変えない**。transitions のテストにフィールドの set/clear を追加
- fixtures は既存の実採取ペイロードをそのまま使う(新規採取は不要)

**アプリ側**(`NotificationManager` / `NotificationLogic`):

- **フォアグラウンド抑制を撤廃**: 3 秒遅延と「配信直前に unreviewed がまだ立っているか」の再チェックは
  残す。フォーカスの有無では抑制しない
- **通知内容**: waiting = タイトル「Waiting for you — \<ラベル\>」、サブタイトルにフックの `message`
  (例: "Claude needs your permission")、本文にそのエージェントの `task`(最後に打った指示)。
  done = タイトル「Done — \<ラベル\>」、本文に `last_assistant_message`(無ければ `task`)
- **完了音**: done 通知にも `.default` サウンドを付ける。Mute Sound は waiting / done 両方に効く。
  interruptionLevel は現行のまま(waiting = time-sensitive、done = active)
- **配信済みバナーの掃除は現行仕様を維持**(新規実装ではない): unreviewed 低下・focus 時の掃除、
  および `agent_removed` 時の掃除(reason == session_end を除く、の例外を含めて §4.5 のまま)。
  抑制撤廃・内容変更の後も掃除の挙動が退行していないことをテストで確認する
- `NotificationLogic` の判定テストを新仕様に合わせて更新(弱めるのではなく、新仕様を表に)

## T5. Setup Check(doctor の GUI 化)

- **CLI**: `shiibar-cc doctor --json` を追加(`crates/shiibar-cc/src/doctor_cmd.rs`)。
  チェック結果を `{"checks":[{"id","status"("ok"|"warn"|"fail"),"summary","hint"}]}` で出力。
  人間向け出力と exit code の意味は現行のまま。判定ロジックは今後も CLI 側が正(アプリに複製しない)
- **アプリ**: `⌄` メニューに「Setup Check…」を追加し、ウィンドウを開く(LSUIElement アプリなので
  表示時に `NSApp.activate` が必要)。内容は: CLI doctor --json の結果 + アプリでしか取れない 2 項目
  (通知許可の状態 / ログイン時起動の登録状態)を ✓/⚠/✗ の一覧で表示。各行に対処のヒント一言。
  Re-run ボタン付き
- 将来ドロップ導入(配布)した場合のオンボーディング画面を兼ねる想定 — 「未設定を検出して直し方を
  示す」構成にしておく(hooks 未設定なら settings-snippet の案内、など)

## T6. uninstall.sh に `--purge` を追加

- 引数なしの挙動は現行のまま(アプリ・symlink・report.sh のみ。入れ直しが楽な撤去)
- `--purge` で追加実行:
  - `~/.claude/settings.json` から shiibar の hooks エントリを除去(`report.sh` を指す command で特定し、
    他の hooks は保持)。書き換え前に `settings.json.bak` へバックアップ
  - state dir(`SHIIBAR_CC_STATE_DIR` 既定パス)削除、`defaults delete cc.shiibar.menubar`
  - `security delete-certificate -c shiibar-cc-local-signing`
  - `tccutil reset AppleEvents cc.shiibar.menubar`
  - 通知許可はプログラムから消せないため、システム設定 → 通知 からの手動削除を案内して終了
- jq の除去フィルタはフィクスチャ的な入力(snippet をマージ済みの settings.json 例)で動作確認する
- `docs/DEVELOPMENT.md` の uninstall 手順の記述を 2 段階(通常 / --purge)に更新する

## T8. トレイアイコン確定デザインの実装 + 実機調整 — 最後・所有者同席

デザインは所有者決定済み(2026-07-05): **✳ 紋章・枠あり**。

- 構図: フルハイト枠(rect (0.8, 0.8, 16, 16.2)、r 3.5、lw 1.4。canvas 20×18)+ 左上に
  **✳ 紋章**(素の U+2733、中心 (5.4, 11.4)、9.5pt 起点)。**プロンプト ❯ と idle の `_` は廃止**
- 状態表示(右下): waiting `!`(中心 (12, 8.6)、10.5pt bold)/ working = **点灯式 4 コマ**
  (全消灯 → `.` → `..` → `...`。ドット y=5.0・dx ±2.4・r 1.05、薄ドット 30%、間隔 500ms 起点、§9)/
  idle = 無印。減光: idle 0.8 / 切断・エージェントなし 0.45(✳ と枠のみ)。
  未確認ドットは枠の角 (16.2, 15.2) r 2.2(ハロー付き・従来の二層規則のまま)
- タイマーは **working 表示のときだけ**動かし、working がいなくなったら停止
  (通常時は従来どおり状態変化時の再描画のみ)。ロールアップ優先順位(waiting > working > idle)は不変
- 実機調整項目: ✳ の光学位置・サイズ / 線の滲み / バッジのかかり具合 / アニメ間隔(400–800ms)と
  薄ドット濃度(25–40%)/ idle 0.8 と切断 0.45 の見え分け。
  dev-reload での実機確認は着手前に手順を先出しして所有者の合図を待つこと
- `docs/menubar-design.html` トレイ節・デザイン原則は確定デザインに反映済み。実機で数値が動いた場合の

## T9. ドロップダウン再設計(フラット一覧 + 状態記号 + 並び替え)+ T2 フォローアップ

所有者決定済み(2026-07-05)。仕様は DESIGN.md §3.6 / §4.2 / §4.5、見た目は menubar-design.html の
ドロップダウン節(いずれも反映済み)。実装順は T5〜T7 より先でよい。

**daemon / proto 側**:

- `AgentEntry` に `created_at`(登録時刻・不変)と `last_report_at`(hook report ごとに daemon の
  時計で更新。reconcile・stale sweep では更新しない)を追加(§3.6)。`#[serde(default)]` で
  旧 state.json を読めること(欠損時の既定は 0 で構わない — 並びの末尾に沈むだけ)
- ワイヤ `Agent` に前方互換で追加(§4.2)。`last_report_at` の変化は `status_changed` の配信対象、
  `created_at` は不変。§3.4 の遷移テーブルと既存テストの期待値は不変。追加フィールドのテストを追記

**アプリ側**:

- 並び 3 モード(§4.5): Newest session(既定・`created_at` 降順)/ Recent activity
  (`last_report_at` 降順・並びの確定は dropdownOpenedAt = 開いている間は動かさない)/
  Grouped(従来のグループ表示。行には状態記号を付けたまま)。⌄ メニューに「Sort by」ラジオ 3 択、
  選択は UserDefaults 永続。フラット 2 モードでは未読の上寄せをしない
- 行頭の状態記号: 空◯(55%)= idle / ◯! = waiting / 回転スピナー(1.7 秒/周)= working。
  スピナーはドロップダウンが開いている間のみ回す
- **未確認バッジを状態記号の右肩へ移動**(赤 + 明色ハロー)。行右端の赤ドットは廃止。太字は維持
- 並び替えの比較ロジック・記号の選択ロジックは ShiibarCcCore の純ロジックとしてテスト

**T2 フォローアップ(⌄ チップ)**:

- ⌄ チップのホバーから選択色(青)を撤去し、**常時グレーのチップ**(`rgba(127,127,127,.14)` 相当・角丸 7px)
  + ホバーで濃く(`.22` 相当)+ プレスでさらに濃く(`.30` 相当)に変更。文字色は変えない
  (menubar-design.html のホバー/プレス項に反映済み。行の選択色ホバーはそのまま)

所有者決定済み(2026-07-05): 黒塗りタイル + 左上 ✳ + 右に状態記号 3 つの縦積み(枠線なし)。

  - 記号列 x=72、r=8、線幅 2.6: ◯+`!`(cy 26。太字、r×1.75 のフォント)+ 赤バッジ
    (`#f2464d`、r 4.2、白 90% のハロー、記号の右肩 +0.74r/−0.74r)/ 開いた円弧(cy 52。270°、
    矢頭なし、round cap)/ 空◯(cy 78。線 50% 濃度)

- **install.sh**: 生成スクリプトを実行して `Contents/Resources/AppIcon.icns` を配置し、
  Info.plist に `CFBundleIconFile`(`AppIcon`)を追加

- 検証: install.sh のドライ実行はしない。スクリプト単体で `.icns` が生成でき、`file` で
