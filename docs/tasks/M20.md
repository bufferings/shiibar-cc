## 背景とゴール

Claude Code は plugin.json の version が変わらない限りキャッシュした plugin を使い続ける
(公式ドキュメント code.claude.com/docs/en/plugin-marketplaces.md の version resolution 節で確認済み)。つまり hooks の変更は version bump がゲート、そして bump しても
利用者側の marketplace auto-update 設定(サードパーティは既定オフ)次第では届かない。

これを 2 つの仕組みで閉じる(仕様は DESIGN.md に反映済み):

1. **bump = ゲート**: plugin.json の version はリリースコミットでタグと同じ番号に上げる。
   `check-version.sh` が tag = Cargo.toml = plugin.json の三点一致を検査してリリースを止める
2. **postflight = 配達**: brew upgrade の postflight で、プラグインが**有効**なら
   `claude plugin marketplace update shiibar-cc` を実行する(アプリと hooks が同時に揃う)

### T2: cask postflight の分岐拡張(`packaging/homebrew/shiibar-cc.rb`)

- 現在: `enabledPlugins` に `shiibar-cc@shiibar-cc` のエントリがあれば(値を問わず)exit 0
- 変更後の 3 分岐:
  1. エントリなし → 従来どおり `marketplace add` + `plugin install`(変更なし)
  2. エントリあり・**値が `true`** → `"$claude_bin" plugin marketplace update shiibar-cc || true`
  3. エントリあり・**それ以外**(`false` 等)→ 何もしない(exit 0)
- 判定は既存と同じ grep ベースで(jq 非依存)。`scripts/dev-install.sh` に `true` 値を
  grep する類例がある
- best-effort の性質は維持: どの分岐の失敗も cask install/upgrade を落とさない
- caveats の文面を新挙動に合わせて更新(「有効なら upgrade 時に hooks も更新される」旨を追記。英語)

### T3: 検証

- **配達コマンドの実効を実測で確定する**: CLI の実物 help では `plugin marketplace update` は
  マーケットプレイス(= リポジトリのクローン)を更新するコマンドで、インストール済み plugin の
  更新は `claude plugin update <plugin>`(restart required)という**別コマンド**として存在する。
  `marketplace update` だけでインストール済み hooks が新しくなるかは未検証。ローカルパスの
  marketplace + 偽 HOME + **実物の claude CLI** で「どのコマンド(列)で hooks が実際に
  更新されるか」を確認し、cask の有効時分岐を実測で確定した形に直す
- **postflight は偽 HOME + スタブ `claude` で実測する**(M18 の brief に同じ手法の記録がある。
  スタブは呼び出された引数をファイルに記録するシェルスクリプトでよい)。最低 4 ケース:
  1. エントリなし → `marketplace add` + `install` が呼ばれる
  2. エントリ `true` → `plugin marketplace update shiibar-cc` **だけ**が呼ばれる
  3. エントリ `false` → claude が一切呼ばれない
  4. claude 不在(スタブなし)→ 何も起きず正常終了
- `check-version.sh` の実測: タグ・Cargo.toml・plugin.json の一致 / plugin.json だけ不一致、の
  2 ケースを一時コピーで実行して exit code を確認(実ファイルの version は動かさない —
  現状 Cargo.toml も plugin.json も 0.1.0 で一致している状態を保つ)
- ci.yml の cask レンダリング + `brew style --cask` が引き続き通ること
- `cargo test --workspace --locked` / `cargo clippy --all-targets --locked` 全緑(Rust 変更なしの確認)
- 実機の brew install/upgrade 経路の検証は v0.1.0 リリース時(M18 と同じ扱い)
