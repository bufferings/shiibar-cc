## 成果物

### 0. daemon: `agent_removed` に `reason` フィールドを追加(DESIGN.md §4.2)

アプリの通知掃除(§4.5)が削除の由来を必要とするため、先にこれだけ Rust 側を変更する:

- `agent_removed` イベントに `reason` を付ける: `"session_end"`(hook)/ `"stale"` / `"remove"`(手動)/
  `"prune"`(reconcile)。proto の型と daemon の全削除経路(§3.6 の 3 経路 + remove コマンド)を更新
- ワイヤ上は前方互換の追加フィールド(ビルド済みクライアントは壊れない)。既存テストの期待値に
  reason を追加し、経路ごとに正しい reason が出ることをテストする
- `reason` は `#[serde(default)]` 付きで追加する(reason なしの既存 JSON を読めること)。
  `crates/shiibar-cc-client/src/wait.rs` の `AgentRemoved { target }` パターンは全フィールド分解のため
  コンパイルが通らなくなる — **`{ target, .. }` への 1 行変更のみ例外として許可**。
  それ以外では M3 が触る crates(shiibar-cc / shiibar-cc-client)に変更を加えないこと

### 1. `app/` — SwiftUI メニューバーアプリ(DESIGN.md §4.5、見た目は menubar-design.html)

Swift Package(executable、macOS 13+)。`MenuBarExtra` の **window スタイル**(ドロップダウンはカスタムビュー)。

- **daemon 接続**: `NWConnection` で UDS(state dir の `shiibar-ccd.sock`。`SHIIBAR_CC_STATE_DIR` 上書き、
  既定 `~/.local/state/shiibar-cc/`)に接続し、`{"cmd":"subscribe"}` を送って行 JSON を `JSONDecoder` で読む
  (行バッファリングを自前で行う)。切断時は 1 秒から倍々・上限 30 秒のバックオフで再接続し、snapshot で回復。
  未知の event / status / フィールドは無視(前方互換。§4.2)
- **daemon ライフサイクル**: 起動時に socket へ接続し、応答があれば既存 daemon にアタッチ。
  応答がなければ同梱の `shiibar-ccd`(`.app` の `Contents/Helpers/`、バンドルからの絶対パス)を spawn して
  バックオフ再接続。Quit 時に `{"cmd":"shutdown"}` を送って daemon も止める。
  開発中(`.app` 化前の `swift run`)は手動起動した daemon へのアタッチで動作確認する(spawn は .app でのみ検証)
- **reconcile の実行**(§4.5): **起動時 / daemon 再接続時(snapshot 受信後)** と、⌄ メニューの Rescan の
  2 経路で `shiibar-cc reconcile` を subprocess 実行する。定期ポーリングはしない(§8.10)
- **トレイアイコン**(menubar-design.html「トレイ」節・実装ノート):
  - ロールアップ: waiting `!` > working `✳` > idle `_`、エージェントなしは `❯` のみ
  - **二層合成**: 窓 + `❯` + 状態文字のテンプレート層の上に、赤ドット(unreviewed が 1 台以上)を別レイヤーで重ねる。
    単一 NSImage に赤を描いてテンプレート指定すると赤が失われる。一次案 = `MenuBarExtra` label で
    ZStack(テンプレート NSImage + 赤 Circle)、不可なら effectiveAppearance 監視の非テンプレート合成
  - 減光 2 段階: idle = 0.8 / エージェントなし = 0.45(実機調整可)。**daemon 切断中はエージェントなしと同じ減光**
  - 赤ドットは右肩オーバーハング + 明色ハロー常時描画。`✳` は U+2733 系の単純な米印に留める
    (Anthropic ロゴのサンバースト形状に寄せない)
- **ドロップダウン**(menubar-design.html「ドロップダウン」節):
  - 最上部 ⌄ メニュー: Back(`shiibar-cc focus -`)/ Rescan(`shiibar-cc reconcile`)/
    Mute Sound(ミュート中チェックマーク)/ Quit。v1 の topbar は ⌄ のみ(Filter 欄は作らない)
  - 警告行(⌄ 直下に常設・モノクロ + ⚠・赤不使用・クリック動作なし): daemon 切断中 /
    通知権限 denied / focus が TCC エラー(exit 3)を返した
  - グループ見出し Waiting / Working / Idle(窓アイコン + 太字。空グループ非表示)、inset カード、
    行 = 1 行目 作業内容(waiting は `message`、他は `task`、無ければラベル昇格)/ 2 行目 ラベル · 経過時間。
    未読 = 1 行目太字 + 赤ドット、focus(seen)で消える。並びは Waiting → Working → Idle・各グループ内は未読が上。
    行クリックで `shiibar-cc focus <target>` を subprocess 実行。`agent_removed` で行を消す。
    エージェントゼロは topbar + 「No agents」のみ。1 行目は末尾省略、多いときは縦スクロール。UI 文言は英語
  - ラベル: cwd をホーム配下なら `~` 起点にし末尾 2 要素(足りなければあるだけ)。重複はそのまま
- **デスクトップ通知**(§4.5): `UNUserNotificationCenter`。unreviewed の立ち上がり(false→true)で発火
  (snapshot / reconcile 経由も含む。同じ立ち上がりの二重通知は発火済み記録で防ぐ)。
  waiting = トースト + 音(time-sensitive)/ 完了(idle+unreviewed)= トースト・音なし。
  同一 target で `threadIdentifier` グループ化。ミュート(UserDefaults)は音だけ止める。
  **遅延通知**: 3 秒後の発火時点で最新状態を再確認し、まだ unreviewed かつ対象が前面でなければ通知
  (前面判定は `shiibar-cc focused` を叩く)。通知クリックは delegate で受けて focus。
  掃除: focus・unreviewed 低下時に該当 target の配信済み通知を消す。
  **`agent_removed` の `reason` が `session_end` のときは消さない**(それ以外の reason は消してよい)
- **subprocess の呼び先**: `.app` 実行時は同梱の `shiibar-cc`(絶対パス)。開発中は PATH の `shiibar-cc` でよい
  (この差はヘルパー解決関数 1 箇所に閉じ込める)

### 2. `scripts/install.sh` / `dev-reload.sh` の M4 拡張(DESIGN.md §4.5 同梱方針・§5)

- install.sh: `swift build -c release` → `.app` バンドル生成(bundle identifier 付き Info.plist。通知に必須)→
  `shiibar-ccd` / `shiibar-cc` を `Contents/Helpers/` に同梱 → **安定した署名 ID で ad-hoc 署名**
  (再ビルドで通知権限がリセットされないように)→ `~/.local/bin/shiibar-cc` を同梱バイナリへの
  シンボリックリンクに置き換え → Login Items 登録。既存の M2 部分(hooks 案内)は壊さない
- dev-reload.sh: アプリの再ビルド・差し替えに対応(開発時の daemon 手動起動の前提は §4.2 運用のまま)
- uninstall.sh: `.app`・Login Items・シンボリックリンクの除去を追加
