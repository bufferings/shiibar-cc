## 背景

Conversations の下部パネルは、live の行で Resume を disabled にして行き止まりだった。
これを 3 状態にする(§4.6 / §8.48): past = Resume / live で Agents 一覧に session_id 一致 =
**Jump**(そのタブへ focus)/ live で一致なし = disabled の Jump。メニューバー UI・CLI・
状態機械には手を入れない。

## タスク

### T1: ボタン状態の導出(純関数、ShiibarCcCore)

- `ShiibarCcCore` に、選択中の会話とその時点の Agents 一覧からボタン状態を導出する純関数を置く
  (`ResumeTerminal.swift` と同じ構成: 判断は Core の純関数・IO と表示はアプリ側)。
  状態は 3 つ + 既存エッジ 1 つ:
  - past(live でない)かつ cwd あり → **Resume(有効)**
  - past かつ cwd 無し → **Resume(disabled)**(既存の `canResume` の挙動を保存 — 変えない)
  - live かつ Agents に session_id 一致あり → **Jump(有効)** — 一致したエントリの target を持つ
  - live かつ一致なし → **Jump(disabled)**
- 一致は `Agent.sessionId` の完全一致。複数一致は仕様上想定しない(session_id は一意)が、
  起きたときは最初の一致でよい(発明しない・防御を足さない)

### T2: 下部パネルと Jump 動作(アプリ)

- 下部パネルを 3 状態表示にする(§4.6):
  - ボタンラベル: `Resume` / `Jump`。**同幅固定**(長い方 = Resume の幅に揃える。
    行を移ってボタンが替わっても端の位置が動かないこと)
  - 注記(ボタンのすぐ左隣・淡色): Jump(有効)には `This conversation is running`、
    Jump(disabled)には `Running in a terminal Shiibar CC can't drive`。
    past の行に注記は無し。ツールチップは付けない
  - disabled の見た目は標準の灰色の器(§8.42。既存の disabled Resume と同じ)
  - パネルの高さは全状態で不変(既存どおり)
- **導出のタイミングは行の選択時と一覧(search 結果)の配達時だけ**(§4.6 / §8.34):
  導出結果を保持し、`AppState.agents` の変化で表示中のボタンを再導出**しない**。
  ビューが agents をライブに観測してボタンが勝手に替わる実装は不可
- **Jump の動作**: 導出時に得た target で `AppState.focus(target:)` を呼ぶ
  (§4.5 の行クリックと同一経路 — exit 2/3 の扱い・成功時の seen 送信は既存のまま)。
  ウィンドウは閉じない。**search の再実行はしない**(Resume 成功後の再実行とは違う — §8.48)

## テスト要求

- T1 の導出関数: 表駆動で全状態(past+cwd / past+cwd 無し / live+一致 / live+一致なし)+
  一致の照合が sessionId 完全一致であること
- T2 の導出タイミング: 「agents の変化だけでは表示状態が変わらない」「選択の切り替え・
  一覧の配達で再導出される」をビューモデルの単位テストで固定(既存の
  `ConversationsViewModel` 系テストの流儀に合わせる)
- Jump が `focus(target:)` に導出時の target を渡すことをスパイで確認(実 CLI・実 osascript は
  起動しない — 既存のテスト分離のまま)
- UI 文言は英語。`docs/` 以外に日本語を書かない。実マシンのパスをコード・テストに入れない

- 所有者スモーク(実機):
