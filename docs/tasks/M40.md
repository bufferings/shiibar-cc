## 背景

Terminal.app を使う実利用者向けに、iTerm2 と同等の機能(追跡・focus・Resume・focused)を
Terminal.app にも提供する。技術的な裏付けは 2026-07-16 のスパイクで実測済み(§7-7)。
このマイルストーンで target が全種 prefix つき(`iterm2:<UUID>` / `apple-terminal:<tty パス>`)に変わる(§2)。
状態機械(§3.4)・プロトコル(§4.2)・メニューバー UI には手を入れない。

## タスク

### T1: target 生成の 2 分岐(proto / report)

- `crates/shiibar-cc-proto` の抽出(`extract.rs`)と `shiibar-cc report` の target 生成を §4.1 の
  2 分岐に更新する。iTerm2 分岐は判定条件(`TERM_PROGRAM` + `ITERM_SESSION_ID` の同時チェック)を
  変えず、生成する target に `iterm2:` を前置するだけ
- `Apple_Terminal` 分岐: 制御端末は**自プロセスからは取れない**(§7-7 — hook は `??`)ので、
  親方向に最初に制御端末を持つプロセスまで辿る(上限 10 段。§9)。取れなければ drop。
  実装は subprocess の `ps` 連打ではなく sysctl 系(`kinfo_proc` の ppid と tty デバイス)を推奨 —
  hooks の 1 秒タイムアウト(§4.1)の中で走る経路であり、リポジトリには `KERN_PROCARGS2` の
  sysctl 前例がある(§7-6)。走査は `TERM_PROGRAM == "Apple_Terminal"` のときだけ行う
  (iTerm2 利用者に新しいコストを掛けない)
- tty は `/dev/ttysNNN` の絶対パス形式で target に入れる(§2)

### T2: state の保存形式移行(daemon)

- daemon の state 読み込みで、`:` を含まない target に `iterm2:` を前置する(§2。1 回きりの
  形式移行 — daemon が prefix の意味を解釈する箇所は作らない)
- 既存テスト・fixtures の期待値を prefix つき表記に更新する(§3.4 の表駆動テストは
  遷移の内容に触らず、target 文字列の表記だけが変わる)

### T3: apple_terminal モジュールと iterm モジュールの改名(client)

- **モジュール名は target の prefix に揃える**(§4.3): 既存の `iterm.rs` を `iterm2.rs` に、
  `iterm_targets` を `iterm2_targets` に改名する(機械的改名。参照・テストも追従。
  §6・§8 の歴史記述に出る旧名は改名しない)
- `crates/shiibar-cc-client/src/apple_terminal.rs` を新設し、§4.3 の 4 関数を実装する:
  `focus` / `focused` / `apple_terminal_targets` / `open_resume_window`。
  AppleScript 生成と出力パースは純関数に切り出し、osascript 実行部は注入可能に(iterm.rs と同じ構成)
- focus は全 windows × 全 tabs 走査(AppKit タブは別 window に見える — §7-7)。
  is-running ガード(`application id "com.apple.Terminal" is running`)必須。
  TCC エラーは「該当なし」と区別(exit 3 系統)
- target の prefix で同名のモジュールに振り分ける入口を 1 箇所作る。**振り分けの規則
  (未知 prefix・`:` 無し・`wNtNpN:` の互換を含む)は §4.3 が正** — 挙動を発明しないこと。
  trait 等の抽象層は作らない(§8.2・§8.47)

### T4: reconcile の合流(client)

- gather の target 導出を §3.5 に合わせる: `pid → tty` を、起動中の対応ターミナルそれぞれの
  走査結果と突き合わせる。未起動のターミナルは走査しない(= 空集合)。
  `ps` の tty(`/dev/` 無し)と AppleScript の tty(`/dev/` 付き)の正規化に注意(§7-7)
- `complete` の意味は「**試みた**走査がすべて成功」(§3.5)。未起動によるスキップは不完全ではない

### T5: CLI(shiibar-cc)

- `resume` に `--terminal iterm2|apple-terminal`(省略時 `iterm2`)を追加(§4.4)。
  daemon 非接続の原則は維持 — 判断はしない、渡された値で開くだけ
- `focused` を両ターミナル対応にする(§4.4): 前面アプリが iTerm2 / Terminal.app のどちらかの
  ときだけ、そのアプリの前面セッションの target(prefix つき)を出力
- `doctor` の TCC probe を「起動中の対応ターミナルごと」にする(§4.4)。未起動はスキップし
  `[info]` で明示。JSON では `ok` に畳む(既存の iTerm2 未起動と同じ扱い)

### T6: Resume の端末決定(アプリ)

- Conversations の Resume ボタンが `--terminal` を渡す(§4.6): エージェント一覧の最新エントリ
  (`last_seen` 最大)の target prefix → 無ければ UserDefaults に記憶した最後の種別
  (エントリを観測するたび更新)→ それも無ければ `iterm2`
- 設定 UI・初回質問は作らない(§8.47)

### T7: fixtures(Terminal.app の実キャプチャ)

  実採取後、`$HOME` 等の実パスはニュートラルな値に置換すること(CLAUDE.md のポータビリティ規則)
