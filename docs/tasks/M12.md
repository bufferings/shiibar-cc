## 背景とゴール

所有者要望(経緯は DESIGN.md §8.24): 未読バッジの一括クリアと、アプリ情報(名前・アイコン・
バージョン)の確認先を ⌄ メニューに追加する。

## タスク

### T1: CLI `shiibar-cc seen <selector>`(`crates/shiibar-cc`)

- daemon に `seen` リクエスト(§4.2 に既存)を送るサブコマンドを追加。selector の解決・
  exit code は既存サブコマンド(`remove` が最も近い)と同じ規約(§4.4: 0 成功 / 1 接続エラー・
  複数一致 / 2 該当なし)
- テスト: `remove` の既存テスト(`tests/exit_codes.rs` ほか)と同じ粒度で、成功・該当なし・
  daemon 不在の exit code を検証

### T2: ⌄ メニュー Clear badges(`app/`)

- Rescan の直後に **Clear badges** 項目を追加(実行項目 = クリックで閉じる。§4.5)
- **未読(unreviewed)が 1 つも無ければ disabled**
- クリック時: その時点の未読 target ごとに CLI `seen <target>` を呼ぶ(`CLIRunner` に追加。
  既存の focus/reconcile と同じ呼び出しパターン)。フィードバック表示は不要 — バッジが消えるのが
  フィードバック(subscribe 経由で即時反映される)
- 通知センターの配信済み通知は**触らない**(§8.24)

### T3: ⌄ メニュー About(`app/`)

- Settings サブメニューの後に **About Shiibar CC** 項目を追加(実行項目)
- クリック時: `NSApp.activate` してから `orderFrontStandardAboutPanel` を呼ぶ
  (LSUIElement のため activate が必要 — Setup Check ウィンドウと同じ扱い。§4.5)。
  パネルの内容(アイコン・名前・バージョン)は bundle の Info.plist から自動で載るため、
  カスタム UI は作らない

### T4: 検証

- `(cd app && swift build && swift test)` 全緑(メニュー構成のテストが既存にあれば追従)
- `cargo test --workspace --locked` / `cargo clippy --all-targets --locked` 全緑・警告ゼロ
- `docs/menubar-design.html` にメニュー構成のモックがあれば、項目追加を反映する
  (数値・見た目の変更はないので、テキストの項目列だけ)
- 実機確認(disabled 表示・About パネル・バッジ一括クリア)は所有者側で行う
