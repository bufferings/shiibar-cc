## 背景とゴール

release.yml のジョブ内には公証キー(.p8)や一時キーチェーンがディスク上に存在する時間帯があり、
そこで動く action はどれも同じ権限で任意コードを実行できる。`@v5` のようなタグ参照は上流で
動かせるため、action の乗っ取りで中身が差し替わる攻撃面が残る。DESIGN.md §10(サプライチェーン
対策)の精神に沿い、全 workflow の `uses:` を**フルコミット SHA 固定**にする。

## タスク

### T1: 3 workflow の `uses:` を SHA 固定にする

`.github/workflows/ci.yml` / `release.yml` / `bump-cask.yml` の全 `uses:` を
`owner/action@<40 桁 SHA> # vX` 形式(SHA 固定 + 元タグをコメントで併記)にする。

使う SHA(2026-07-08 に GitHub API `repos/<owner>/<repo>/git/ref/tags/<tag>` で解決済み。
annotated tag は commit まで剥がした値。**鵜呑みにせず、適用前に同じ方法で再確認すること**):

- `actions/checkout@v5` → `93cb6efe18208431cddfb8368fd83d5badbf9bfd`
- `actions/upload-artifact@v4` → `ea165f8d65b6e75b540449e92b4886f43607fa02`
- `EmbarkStudios/cargo-deny-action@v2` → `bb137d7af7e4fb67e5f82a49c4fce4fad40782fe`

対象は現時点で: ci.yml の `actions/checkout` ×2 と `EmbarkStudios/cargo-deny-action` ×1、
release.yml の `actions/checkout` ×1 と `actions/upload-artifact` ×1、bump-cask.yml の
`actions/checkout` ×2。漏れがないよう `grep -n "uses:" .github/workflows/*.yml` で全数を確認する。

### T2: DESIGN.md §10 に方針を 1 行追記する

(タグはコメントで併記)。secrets を扱う workflow を持つため」の趣旨を 1 項目追加する。
