## 背景とゴール

iTerm2 から起動した VS Code の統合ターミナルは `$ITERM_SESSION_ID` を環境変数ごと相続する
(起動元タブの UUID を指す)。現行の「`ITERM_SESSION_ID` があれば track」だと VS Code 内の
Claude Code セッションが誤って追跡され、focus すると無関係な iTerm2 タブに飛ぶ(2026-07-05
実機で確認。詳細は DESIGN.md §7-1)。

修正: DESIGN.md §4.1 の新しい生成規則のとおり、**`TERM_PROGRAM == "iTerm.app"` かつ
`ITERM_SESSION_ID` あり**のときだけ target を生成し、どちらかを欠けば report を drop する。
