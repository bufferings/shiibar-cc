デザイン原則

  • 色は赤ただひとつ = 「あなた宛の未読(unreviewed)」。通知の発火条件と 1:1。 status は色ではなく記号と濃度で表す — !(waiting・太)/ 点灯ドット(working・4 コマアニメ)/ 無印(idle・全体 0.8)
  • Claude が居る窓: トレイ = フルハイトの角丸窓 + 左上の ✳ 紋章 (Claude Code のスピナー文字)+ 右下の状態表示。 「ターミナルで Claude Code が動いている姿」(✳ の下で ... が進むあの画面)をそのまま記号化したもの
  • 太さと濃度の階層が色の代役: ! は太く、点灯ドットは濃淡で動き、 idle・切断は全体の減光で沈む。グレースケールだけで優先順位が読める(色覚特性にも強い)
  • アイコンは操作ボタンを描かず、状態そのものを描く(▶/⏸ のようなメディアボタン文法は逆読みされるため不採用)

実装ノート

  • トレイは二層合成: 窓 + ✳ + 状態表示のテンプレート層(モノクロ・自動追従)の上に、 赤ドットを別レイヤーで重ねる。単一 NSImage に赤を描いてテンプレート指定すると赤が失われる点に注意。 一次案 = `MenuBarExtra` の label で ZStack(テンプレート NSImage + 赤 Circle オーバーレイ)、 不可なら effectiveAppearance 監視の非テンプレート合成にフォールバック。 ✳ 紋章と ! は文字グリフ(attributed string)、点灯ドットとバッジは自前描画
  • ✳ は 8 本スポークの単純な米印(U+2733 系の文字記号)に留め、Anthropic ロゴのサンバースト形状には寄せない(ブランド流用回避。公開リポジトリなので特に)
  • ドロップダウンは標準 NSMenu ではなくカスタムビュー(SwiftUI `MenuBarExtra` の window スタイル)。 ホバー・クリック・警告行は自前実装になる(M4 の工数に織り込む)
  • 見出しラベルはアイコンの縦センター基準 + わずかに下げる光学調整(モックでは 1.5px 相当)
  • 線の太さ・間隔・光学調整はすべて実機(Retina レンダリング)で最終決定。実機確認項目: ✳ と枠の間隔が融合して見えないか / 点灯ドットの濃淡差が小サイズで読めるか / 見出し窓アイコン 24px の可読性(Grouped モード)/ 赤ドットのハローが各種メニューバー色で機能するか
  • トレイのロールアップは「waiting 2 台」と「1 台」を区別しない(意図的な割り切り。内訳は 1 クリックで見える)
