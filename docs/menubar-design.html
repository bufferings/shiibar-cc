デザイン原則

  • 色は赤ただひとつ = 「あなた宛の未読(unreviewed)」。通知の発火条件と 1:1。 status は色ではなく記号と濃度で表す — !(waiting・太)/ 瞬くグリフ循環スピナー(working)/ 静止した ✻(idle・全体 0.8)
  • Claude が居る窓: トレイ = フルハイトの角丸窓 + 左上スロットの 1 グリフ (Claude Code のスピナー文字)。スロットの中身がロールアップそのもの — 「ターミナルで Claude Code が動いている姿」をそのまま記号化したもの
  • 太さと動き・濃度の階層が色の代役: ! は太く、working は瞬くグリフ循環で動き、 idle・切断は全体の減光で沈む。グレースケールだけで優先順位が読める(色覚特性にも強い)
  • アイコンは操作ボタンを描かず、状態そのものを描く(▶/⏸ のようなメディアボタン文法は逆読みされるため不採用)

トレイ: 角丸の窓 + 左上スロットの 1 グリフ(ロールアップ・常に 1 枠)

実装ノート

  • トレイは二層合成: 窓 + スロットグリフのテンプレート層(モノクロ・自動追従)の上に、 赤バッジを別レイヤーで重ねる。単一 NSImage に赤を描いてテンプレート指定すると赤が失われる点に注意。 一次案 = `MenuBarExtra` の label で ZStack(テンプレート NSImage + 赤 Circle オーバーレイ)、 不可なら effectiveAppearance 監視の非テンプレート合成にフォールバック。 スロットの ✻・!・グリフ循環の各コマはすべて文字グリフ(attributed string)、バッジは自前描画
  • ✻ は素の Unicode 文字グリフ(U+273B TEARDROP-SPOKED ASTERISK 系の文字記号)のまま使い、 Anthropic ロゴのサンバースト形状には寄せない(ブランド流用回避。公開リポジトリなので特に)
  • ドロップダウンは標準 NSMenu ではなくカスタムビュー(SwiftUI `MenuBarExtra` の window スタイル)。 ホバー・クリック・警告行は自前実装になる(M4 の工数に織り込む)
  • 線の太さ・間隔・光学調整はすべて実機(Retina レンダリング)で最終決定。実機確認項目: ✻ と枠の間隔が融合して見えないか / working のグリフ循環が小サイズで判読できるか (花弁が潰れる場合は 9.5pt → 10.5pt まで戻してよい) / 赤バッジのハローが各種メニューバー色で機能するか
  • トレイのロールアップは「waiting 2 台」と「1 台」を区別しない(意図的な割り切り。内訳は 1 クリックで見える)
