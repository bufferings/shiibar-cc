Conversations ウィンドウ表示仕様

見た目の正はこのファイル。挙動(索引・検索・状態規則・CLI 契約)の正は DESIGN.md §4.6 / §4.4。 会話の履歴を眺め・探し・読み、必要なら新しい iTerm2 ウィンドウで再開する画面。

デザイン原則

  • 読む画面として組む: この画面の主活動は「自分の短い指示 → Claude の長い返答」の 繰り返しを読むこと。自分の発言は本文ではなく見出し(全幅の帯)、Claude の返答は Markdown を描画した本文として扱う。You / Claude のラベル行は置かない (帯 = 自分、地の文 = Claude で判別できる)
  • 2 ペインのマスター・ディテール: 左で選ぶ、右で読む、右下で再開。 サイドバーは沈んだ色(サイドバー素材)、読む面は明るい背景 — 選ぶ場所と読む場所を色で分ける
  • 表示が変わるのは利用者が動かしたときだけ: 開く・打鍵・⟳・自分の Resume。 勝手に更新されない(DESIGN.md §4.6)
  • 実行中の会話は文字の印で示す: 2 行目に running。Agents ビューの 状態記号(スピナー・!)は使わない — この画面の live 印の鮮度は最後の検索実行時点であり、 「いまの状態」を主張しない。色も使わない(赤 = unreviewed の専用色はこの画面に存在しない)

ウィンドウのクローム: フルハイトサイドバー

  • タイトルバーは非表示、信号機はサイドバーの上に載る(サイドバーの色が ウィンドウ上端まで届く、システム設定と同じ形)。一覧ウィンドウの「信号機の帯」とは ここが意図的に違う
  • サイドバー = サイドバー素材(沈んだ色)/ 右ペイン = 通常のウィンドウ背景。 境界は 1px の区切り線
  • 一覧の行にホバーのハイライトを出す(ドロップダウンと同じ作法)。行間の罫線・ グループ見出しは置かない — 区切りはホバーと行の内部構造(下記)で読ませる

browse(空クエリ): 最近の会話を眺める

  • 開いた直後の姿。全会話が新しい順(実行中は running 印付きで自然に上位に混ざる)
  • 行は 2 行 — 1 行目 = タイトル(無ければフォルダラベル)+ 経過時間(右端・淡色) / 2 行目 = フォルダラベル(+ 実行中なら · running)。 経過時間を 1 行目の右端に置くのは、2 行目を短く保って行のリズムを崩さないため
  • 右は選択中の会話の全文。下端(最新)から上へスクロールして読む
  • 下部パネルは常に出す。Resume が押せるのは past の行だけで、live の行では disabled + This conversation is running の一言(読むだけ)
  • ステータス行は N conversations (M running)

読む面(右ペイン)の文法

  • 自分の発言 = 全幅の帯: 左端から右端までマージンなしで背景の明度だけを変え、 本文と同じ濃さの ❯(モノスペース・セミボールドはこのグリフのみ)を先頭に、 発言は本文と同じウェイトで載せる。 太字にしない(帯が見出しの仕事をするので文字を強めない)。文字サイズは本文 +0.5pt。 帯の直前は広めの余白(節の切れ目)。スクロールすると帯の列 = 「何を頼んだか」の目次として 流し読みできる
  • Claude の返答 = Markdown を描画(描画範囲は DESIGN.md §4.6): コードブロックはモノスペース + 淡い背景 + 横スクロール、インラインコード・強調・見出し・ 箇条書きも描画
  • パイプ表はグリッドで描画: ヘッダ行はセミボールド + 淡い背景、行間に罫線、 全列左揃え(列揃え指定は無視)。セルの中のインライン(コード・強調)も効く。 表は内容幅のまま(本文幅いっぱいに伸ばさない)。セルは幅の上限(約 28em)で 折り返し、それでも溢れる表だけコードブロックと同じく表の中で横スクロール (DESIGN.md §8.38)。等幅整形の器に載せる形は採らない — 日本語のグリフ幅が モノスペースで揃わず、列が崩れる(DESIGN.md §8.37)
  • 選択とコピー: メッセージをまたぐ連続選択ができる。⏺ マーカー・ 展開/畳みボタンなどの操作部品は選択・コピーに含めない。右クリックは Copy(⌘C)/ Copy as Markdown(⇧⌘C)の 2 項目だけ(ショートカット併記・ 選択が無い間は両方 disabled)。Copy = 表示どおり(書式も WebKit 標準のコピーに任せる)、 Copy as Markdown = 選択範囲を Markdown に直列化(選択したものが Markdown として出てくる)。選択が無ければ何もコピーできない(DESIGN.md §4.6)
  • 返答の先頭に ⏺ マーカー(ぶら下げ): 1 つの指示に返答が連発しても境目が 読めるよう、すべての Claude の返答の先頭に淡い二次色の小さな ⏺ を付ける(Claude Code の ターミナルと同じ文法。連発時だけの条件分岐はしない)。マーカーは本文の左にぶら下げ、 折り返しは ⏺ の下に回り込まない。帯の ❯ と ⏺ は同じグリフ列、帯の文字と本文は 同じ列 — 画面全体が「グリフ + 本文」の 2 列グリッドになる。状態の意味は持たない(Agents ビューの状態記号とは別物・色分けしない)。⏺ は描画後テキストに含めない (ヒット計算・畳み境界の対象外 — DESIGN.md §4.6)
  • 読む幅の上限: 本文の行長は約 60em で頭打ち。ウィンドウを広げても行が 間延びしない(コードブロックと、折り返してもなお溢れる表は、この上限の内側で横スクロール)
  • 本文の既定は 13pt・行間 1.7。コードブロックは本文 −1.5pt

search(2 文字以上): ヒットは「どこにあるか」まで見せる

  • 行の形は browse と同じ(スニペットは出さない — 選択すれば即座に全文とヒットが見えるため)
  • 複数語(空白区切り)は AND。ハイライトは全語同色・現在位置のみ濃く(DESIGN.md §4.6)
  • 行を選ぶと、描画後のテキストに対して語の出現をアプリが計算し、本文にヒットが 1 件以上ある ときだけ右ペインの上に検索バー行を出す(0 件なら出さない)。 形は macOS 標準の文書内検索: カウンタ N of M + ‹ › の セグメンテッドコントロール(ツールチップ付き・内容サイズで左寄せ — バーの幅いっぱいに伸ばさない)。次(›・⌘G)= 新しい方へ / 前(‹・⇧⌘G)= 古い方へ。端まで行ったら反対側へ折り返す。 ヒットが 1 件でも押下のたびに現在のヒットへスクロールする。 初期位置は最新のヒット
  • スクロールバーのヒット目盛り: 右端に、会話全体のどこにヒットが散っているかの 目盛りを重ねる(現在位置だけ濃い色)。画面外・畳みの中のヒットの存在がここで見える。 位置はヒットを含むメッセージ単位の近似でよい
  • 畳まれたメッセージ(下記)の見えない部分にヒットがあるときは、展開ボタンに 件数バッジ(N matches・ハイライト色)を付ける
  • 絞り込み中のステータス行は N of M conversations

  • 畳み閾値(500 文字 — DESIGN.md §9)を超えるメッセージは先頭だけ見せて Show full message。見えない部分にヒットがあれば N matches バッジ

実装ノート

  • 右ペインのメッセージ面は WKWebView(DESIGN.md §4.6 / §8.38)。本ページの メッセージ面の CSS(帯・⏺・コードブロック・表・ハイライト)は誌面の例示であると同時に、 実装のスタイルシートの下敷きでもある。ヘッダ・検索バー行・Resume・一覧はネイティブ
  • 初期サイズ 640×480pt。縦横リサイズ可・サイズと位置を記憶。サイドバー幅は境界の ドラッグで可変(初期 250pt・200〜400pt)・幅を記憶(DESIGN.md §9)。境界はホバーで リサイズカーソルに変わり、境界線がわずかに濃くなる(掴めることが見た目で分かる)
  • ヘッダ下の区切り線は検索バーの有無によらず常時。⟳ は実行中 disabled + ステータス行に Refreshing…(DESIGN.md §4.6)
  • running の会話でも下部パネルは常に同じ高さで出す — Resume を disabled にし、 左に This conversation is running を淡く(DESIGN.md §4.6)
  • 検索欄・選択色・ホバーの作法はドロップダウン(menubar-design.html)と共通。 本ページの CSS 値(ハイライト色・帯の明度含む)は誌面上の例示で、実装はシステムの色・素材に追従させる (サイドバーは NSVisualEffectView のサイドバー素材相当)
  • 右ペインのヘッダの 2 行目はホーム相対のフルパス(~/Documents/blog 形式)+ 経過時間 — 行の 2 要素ラベルより詳しくするのは、再開先のフォルダをここで確定できるようにするため
  • メッセージ間のツール実行は何も描かない(DESIGN.md §4.6)
  • 一覧の差分描画・選択・スクロール記憶のキーは session_id(更新でちらつかない・選択が飛ばない)
  • UI 文言は英語。行・本文の中身(タイトル・発話)はユーザーデータそのまま
  • 空状態(会話ゼロ)は「No conversations」、検索 0 件は「No matching conversations」の一行 + ステータス行 0 of M conversations
