Conversations ウィンドウ表示仕様

見た目の正はこのファイル。挙動(索引・検索・状態規則・CLI 契約)の正は DESIGN.md §4.6 / §4.4。 会話の履歴を眺め・探し・読み、必要なら新しい iTerm2 ウィンドウで再開する画面。

デザイン原則

  • 2 ペインのマスター・ディテール: 左で選ぶ、右で読む、右下で再開。 右ペインは会話の全文(下端 = 最新)で、読む活動に幅を割く
  • 表示が変わるのは利用者が動かしたときだけ: 開く・打鍵・⟳・自分の Resume。 勝手に更新されない(DESIGN.md §4.6)
  • 実行中の会話は文字の印で示す: 2 行目に running。Agents ビューの 状態記号(スピナー・!)は使わない — この画面の live 印の鮮度は最後の検索実行時点であり、 「いまの状態」を主張しない。色も使わない(赤 = unreviewed の専用色はこの画面に存在しない)
  • クロームは一覧ウィンドウと同じ: タイトルバー非表示 + 信号機の帯。 行の文法(1 行目 = 主内容 / 2 行目 = ラベル · 経過時間)もドロップダウンと同じ

browse(空クエリ): 最近の会話を眺める

  • 開いた直後の姿。全会話が新しい順(実行中は running 印付きで自然に上位に混ざる)
  • 行は「1 行目 = タイトル(無ければフォルダラベル)/ 2 行目 = フォルダラベル · 経過時間」
  • 右は選択中の会話の全文。下端(最新)から上へスクロールして読む
  • Resume ボタンは past の行だけ。live の行を選ぶとボタンは出ない(読むだけ)
  • ステータス行は N conversations (M running)

search(2 文字以上): ヒットナビゲーション付きの全文

  • 行の形は browse と同じ(スニペットは出さない — 選択すれば即座に全文とヒットが見えるため)
  • 複数語(空白区切り)は AND — 全語を含む会話に絞り込む。ハイライトは全語同色・ 現在位置のみ濃く(DESIGN.md §4.6)
  • 行を選ぶと、取得した全文に対して語の出現をアプリが計算し、本文にヒットが 1 件以上ある ときだけ右ペインの上に検索バー行(文書順カウンタ + ▲ = 古い方へ / ▼ = 新しい方へ)を出す(0 件なら出さない)。初期位置は最新のヒット。 全出現をハイライトし、現在位置は濃く
  • 500 文字を超えるメッセージは畳み(Show full message)。畳み内のヒットもカウンタに 数え、▲▼ でジャンプしたら自動展開
  • 絞り込み中のステータス行は N of M conversations

初回構築中の見え方

  • ウィンドウを開くと `conversations index` の進捗が来る。全構築中は検索欄を無効にし、 ステータス行に Indexing 340 of 4,200…。完了したら一覧が出る(一覧が徐々に埋まる演出はしない)
  • 回る ✻ は使わない — グリフ循環スピナーは製品内で「working」の意味を持つ(menubar-design.html)
  • エラー時: index のエラーは一覧を出さずにエラー表示、search のエラーは前回結果を維持して ステータス行にエラー表示(DESIGN.md §4.6)

実装ノート

  • 初期サイズ 640×480pt(左ペイン 280pt)。縦横リサイズ可・サイズと位置を記憶(DESIGN.md §9)
  • 行・検索欄・選択色・ホバーの作法はドロップダウン(menubar-design.html)と共通。 本ページの CSS 値(ハイライト色含む)は誌面上の例示で、実装はシステムの色・素材に追従させる
  • 右ペインのヘッダの 2 行目はホーム相対のフルパス(~/Documents/blog 形式)+ 経過時間 — 行の 2 要素ラベルより詳しくするのは、再開先のフォルダをここで確定できるようにするため
  • 発話ラベルは You / Claude の 2 種のみ。メッセージ間のツール実行は何も描かない(DESIGN.md §4.6)
  • 一覧の差分描画・選択・スクロール記憶のキーは session_id(更新でちらつかない・選択が飛ばない)
  • UI 文言は英語。行・本文の中身(タイトル・発話)はユーザーデータそのまま
  • 空状態(会話ゼロ)は「No conversations」、検索 0 件は「No matching conversations」の一行 + ステータス行 0 of M conversations
